HTTP - это протокол для получения ресурсов, например, HTML-документов. Он лежит в основе обмена данными в Интернете и является протоколом клиент-серверного взаимодействия, что означает инициирование запросов к серверу самим получателем, обычно веб-браузером. Клиент и сервер взаимодействуют, обмениваясь отдельными сообщениями (а не потоком данных).

Между клиентом и сервером как правило существуют многочисленные посредники, называемые прокси, которые выполняют различные операции и работают как шлюзы или, например, кэш. Это серверное оборудование или программные сервисы, развёрнутые в сети на пути следования пакета данных. Они могут располагаться: в локальной сети организации, у интернет‑провайдеров, в дата‑центрах CDN‑провайдеров, в облачных платформах (AWS, Cloudflare и др.).

HTTP - протокол прикладного уровня, который использует соединение TCP (или TLS-защищённый TCP) для пересылки сообщений, однако теоретически может быть использован любой другой надёжный транспортный протокол (QUIC). Благодаря своей расширяемости, HTTP используется не только для получения гипертекстовых документов и их частей, но и для изображений и видео, а также для отправки содержимого серверам, например, с помощью HTML-форм.

Версия **HTTP/1.1**, выпущенная в 1999 году, работает по принципу «один запрос — одно соединение». Для загрузки страницы с множеством ресурсов (CSS, JS, изображения) браузер открывает несколько TCP‑соединений, что создаёт накладные расходы и увеличивает задержки. Протокол использует текстовый формат, не сжимает заголовки и не поддерживает Server Push.

[[HTTP]] #TLS #SSL

HTTPS — не отдельная версия HTTP, а механизм шифрования (SSL/TLS) поверх HTTP. Он обеспечивает конфиденциальность, целостность и аутентификацию данных: информация передаётся в зашифрованном виде, сервер подтверждается цифровым сертификатом, а любые изменения данных в пути обнаруживаются. HTTPS использует порт 443 и повышает доверие пользователей и ранжирование в поисковых системах.

На практике HTTP/2 заметно ускоряет загрузку страниц (на 15–30 % для сайтов с более чем 10 ресурсами), особенно на медленных сетях. Однако большинство браузеров поддерживают HTTP/2 только через HTTPS. Для внедрения нужны: поддержка протокола на сервере и действующий SSL/TLS‑сертификат. При этом для простых сайтов с малым числом ресурсов переход на HTTP/2 может не дать заметного выигрыша из‑за накладных расходов на шифрование и бинарный парсинг.

HTTPS можно использовать со всеми версиями HTTP. Это не отдельная версия протокола, а шифрование (SSL/TLS) поверх HTTP:

- HTTPS + HTTP/1.1 — работает, но менее эффективен из‑за ограничений HTTP/1.1;
- HTTPS + HTTP/2 — распространённый стандарт: шифрование компенсируется преимуществами мультиплексирования;
- HTTPS + HTTP/3 — наиболее современная комбинация: QUIC уже включает TLS 1.3, обеспечивая шифрование «из коробки».

На практике:

- большинство браузеров поддерживают HTTPS со всеми версиями HTTP;
- для активации HTTPS нужен SSL/TLS‑сертификат и настройка сервера;
- переход на HTTP/2 или HTTP/3 через HTTPS даёт заметный прирост скорости, особенно для сложных сайтов и медленных сетей;
- HTTP/3 пока поддерживается не всеми серверами и провайдерами, но постепенно становится стандартом.

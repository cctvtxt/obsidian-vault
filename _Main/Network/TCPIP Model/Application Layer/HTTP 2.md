HTTP/2, появившаяся в 2015 году, устраняет эти ограничения. Она работает поверх одного TCP‑соединения, используя мультиплексирование: множество запросов и ответов передаются параллельно по одному каналу. Протокол применяет бинарный формат данных, сжимает заголовки алгоритмом HPACK, позволяет серверу отправлять ресурсы до их запроса (Server Push) и даёт возможность задавать приоритеты запросов.

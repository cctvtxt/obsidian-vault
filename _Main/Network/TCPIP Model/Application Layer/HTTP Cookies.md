[[HTTP 1.1]]

**Cookie** - небольшой фрагмент данных, который сервер отправляет в веб-браузер пользователя. Браузер может сохранять куки, создавать новые, изменять существующие и отправлять их обратно на тот же сервер при последующих запросах. Куки позволяют веб-приложениям хранить ограниченный объем данных и запоминать информацию о состоянии. По умолчанию HTTP является протоколом **без сохранения состояния**.

Как правило, сервер использует содержимое HTTP-куки для определения того, поступают ли разные запросы от одного и того же браузера/пользователя, и в зависимости от этого выдает персонализированный или стандартный ответ

Для хранения пользовательских данных рекомендуется использовать современные API для хранения данных, например **Web Storage API** (`localStorage` и `sessionStorage`) и **IndexedDB**. Они разработаны с учетом требований к хранению данных, никогда не отправляют данные на сервер и **не имеют других недостатков, связанных с использованием файлов cookie для хранения данных**.

- Браузеры, как правило, имеют ограничение на максимальное количество файлов cookie для одного домена (в зависимости от браузера, обычно это сотни файлов) и **максимальный размер одного файла cookie (обычно 4 КБ)**
- Файлы cookie отправляются при каждом запросе, поэтому они могут снижать производительность (например, при медленном мобильном интернет-соединении), особенно если у вас установлено много файлов cookie

После получения HTTP-запроса сервер может отправить в ответ один или несколько заголовков `Set-Cookie`, каждый из которых устанавливает отдельный файл cookie. Файл cookie устанавливается с помощью пары «имя-значение»

```http
Set-Cookie: id=a3fWa; Expires=Thu, 31 Oct 2021 07:28:00 GMT;
```

В зависимости от атрибутов, заданных в заголовке **Set-Cookie** при создании файлов cookie, они могут быть постоянными или сеансовыми. Постоянные файлы cookie удаляются после даты, указанной в атрибуте **Expires** или после периода, указанного в атрибуте **Max-Age**. Сеансовые файлы cookie — файлы cookie без атрибутов **Max-Age** или **Expires** — удаляются по завершении текущего сеанса

---

Файл cookie с атрибутом **Secure** отправляется на сервер только в зашифрованном виде по протоколу HTTPS. Он никогда не отправляется по незащищенному протоколу HTTP (кроме как на локальном хосте). На незащищенных сайтах (http) нельзя устанавливать файлы cookie с атрибутом **Secure**.

К cookie-файлу с атрибутом **HttpOnly** нельзя получить доступ с помощью JavaScript, например с помощью `Document.cookie`; доступ к нему возможен только после того, как он попадет на сервер. Например, cookie-файлы, сохраняющие пользовательские сессии, должны иметь атрибут **HttpOnly** — было бы небезопасно предоставлять к ним доступ для JavaScript. Эта мера предосторожности помогает защититься от межсайтового скриптинга (**XSS**).

Атрибут **Domain** указывает, какой сервер может принимать файлы cookie. Если он указан, файлы cookie будут доступны на указанном сервере и его поддоменах. Например, если вы укажете `Domain=mozilla.org`, файлы cookie будут доступны в этом домене и на поддоменах, таких как `developer.mozilla.org`.

Если в заголовке **Set-Cookie** не указан атрибут **Domain**, файлы cookie доступны на сервере, который его установил но не на его поддоменах. Таким образом, указание **Domain** накладывает меньше ограничений, чем его отсутствие. Cервер может установить атрибут **Domain** только для **своего домена или родительского домена, но не для поддомена или какого-либо другого домена**.

Атрибут **Path** указывает на путь к URL, который должен существовать в запрошенном URL, чтобы можно было отправить заголовок Cookie

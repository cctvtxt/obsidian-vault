WS-SecureConversation — это стандарт WS-\* для SOAP, который оптимизирует безопасность при обмене множеством сообщений, устанавливая защищенный сеанс (security context) вместо повторной аутентификации для каждого запроса.

## Установка контекста безопасности

Инициатор (клиент) начинает с отправки SOAP-сообщения с запросом на создание Security Context Token (SCT) через WS-Trust: содержит идентификатор сеанса и мастер-ключ (производный от симметричного или асимметричного обмена). Сервер (Security Context Service) подтверждает, генерирует SCT с сессионными ключами (derived keys via PSK или SP800-108) и возвращает его. Это использует WS-Security для начального обмена, обеспечивая взаимную аутентификацию.

## Обмен сообщениями в сеансе

Последующие SOAP-сообщения ссылаются на SCT в заголовке `wsc:SecurityContextToken`, используя сессионные ключи для подписи и шифрования. Ключи обновляются периодически (key renewal), предотвращая компрометацию долгоживущих сеансов. Поддерживает анонимные или идентифицированные контексты, интегрируясь с WS-SecurityPolicy для конфигурации.

## Преимущества

Снижает криптографическую нагрузку (симметричное шифрование вместо асимметричного на каждый запрос), повышает производительность в чатах или batch-обменах. Обеспечивает конфиденциальность, целостность и replay-защиту в распределенных системах, идеально для enterprise (финтех, B2B)

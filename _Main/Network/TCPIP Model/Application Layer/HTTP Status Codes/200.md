[[HTTP Status Codes]]

##### 2* класс кодов (успешно обработанные запросы). Эти коды информируют об успешности принятия и обработки запроса. Также сервер может передать заголовки или тело сообщений.

200 OK: запрос успешно обработан. Это стандартный ответ для успешных HTTP-запросов.

201 Created: в результате успешного выполнения запроса был создан новый ресурс. Сервер должен вернуть заголовок `Location` с URL нового ресурса.

202 Accepted: запрос принят к обработке, но еще не завершен. Полезно для асинхронных операций, когда результат будет готов позже.

203 Non-Authoritative Information: информация в ответе получена из кэша или прокси, а не от исходного сервера, и может быть устаревшей.

204 No Content: запрос успешно обработан, но в ответе нет тела. Полезно для запросов `DELETE` или `PUT`, когда клиенту не нужно получать данные.

205 Reset Content: сервер предписывает клиенту сбросить представление документа. Например, очистить форму после успешной отправки данных.

206 Partial Content: сервер вернул только часть запрошенного ресурса. Используется для докачки файлов или загрузки больших медиафайлов.

207 Multi-Status: ответ содержит несколько независимых статусов для разных частей запроса. Часто используется в WebDAV.

226 IM Used: сервер выполнил GET-запрос для ресурса, и ответ является представлением результата одного или нескольких манипуляций экземплярами, примененных к текущему экземпляру.

[[HTTP Status Codes]]

##### 4* класс кодов (ошибки на стороне клиента). Эти коды указывают на ошибки со стороны клиентов.

400 Bad Request: сервер не может обработать запрос из-за синтаксической ошибки на стороне клиента. Например, неверный формат JSON или некорректные параметры.
    При использовании метода `PATCH`, эта ошибка часто возникает, если не указан или неверно указан заголовок `Content-Type`. Для `PATCH` с `JSON` необходимо указывать: `Content-Type: application/json`

401 Unauthorized: для доступа к ресурсу требуется аутентификация. Клиент должен предоставить учетные данные в заголовке `Authorization`.

402 Payment Required: зарезервировано для будущего использования. Предполагается для платных сервисов.

403 Forbidden: у клиента нет прав доступа к ресурсу, даже если он аутентифицирован. Отказ в доступе на уровне прав, а не аутентификации.

404 Not Found: сервер не может найти запрошенный ресурс. Самая частая ошибка, обычно вызвана опечаткой в URL или удаленной страницей.

405 Method Not Allowed: используемый метод (например, `POST`) не поддерживается для данного ресурса. Сервер должен вернуть заголовок `Allow` со списком поддерживаемых методов.

406 Not Acceptable: сервер не может предоставить ответ в формате, который запросил клиент через заголовок `Accept`.

407 Proxy Authentication Required: требуется аутентификация на прокси-сервере. Аналогично 401, но для прокси.

408 Request Timeout: сервер не получил полный запрос от клиента в течение установленного времени.

409 Conflict: запрос не может быть выполнен из-за конфликта с текущим состоянием ресурса. Например, попытка создать уже существующий уникальный ресурс.

410 Gone: ресурс был удален и больше недоступен. В отличие от 404, это удаление является преднамеренным.

411 Length Required: сервер требует указания заголовка `Content-Length` для обработки запроса.

412 Precondition Failed: одно из предварительных условий, указанных в заголовках запроса, не выполнено.

413 Payload Too Large: тело запроса превышает максимальный размер, установленный сервером. Ранее назывался "Request Entity Too Large".

414 URI Too Long: URI запроса слишком длинный для обработки сервером. Ранее назывался "Request-URI Too Long".

415 Unsupported Media Type: сервер не поддерживает медиа-тип данных в теле запроса.

416 Range Not Satisfiable: запрошенный диапазон данных не может быть предоставлен. Ранее назывался "Requested Range Not Satisfiable".

417 Expectation Failed: сервер не может удовлетворить ожидания, указанные в заголовке `Expect`.

418 I'm a teapot: клиентский код ошибки, возвращаемый некоторыми серверами в качестве первоапрельской шутки. Не используется в реальных приложениях.

422 Unprocessable Entity: сервер понимает тип контента, но не может обработать запрос из-за семантических ошибок.

423 Locked: запрошенный ресурс заблокирован.

424 Failed Dependency: запрос не может быть выполнен из-за сбоя в выполнении другого, зависимого запроса.

425 Unordered Collection: зарезервировано для WebDAV.

426 Upgrade Required: клиент должен переключиться на другой протокол, например, TLS/1.0, указанный в заголовке `Upgrade`.

428 Precondition Required: сервер требует, чтобы запрос был условным.

429 Too Many Requests: клиент отправил слишком много запросов за определенный промежуток времени. Используется для защиты от DDoS и rate limiting.

431 Request Header Fields Too Large: один или все заголовки запроса слишком велики.

434 Requested Host Unavailable: запрашиваемый хост недоступен.

444 No Response: нестандартный код Nginx, указывающий, что сервер закрыл соединение без отправки ответа.

449 Retry With: нестандартный код Microsoft. Запрос следует повторить после выполнения соответствующего действия.

450 Blocked by Windows Parental Controls: нестандартный код Microsoft. Доступ заблокирован "Родительским контролем" Windows.

451 Unavailable For Legal Reasons: доступ к ресурсу ограничен по юридическим причинам.

456 Unrecoverable Error: неустранимая ошибка.

499 Client Closed Request: нестандартный код Nginx. Клиент закрыл соединение до того, как сервер успел ответить.

[[Session Layer]]

Протокол SSL 3.0 (Secure Sockets Layer) формально определён в документе RFC 6101. Этот стандарт был опубликован для документирования исторической спецификации, которая уже на тот момент считалась устаревшей и небезопасной.

SSL 3.0 — это криптографический протокол, предназначенный для установления безопасного соединения между клиентом и сервером. Его основной задачей было обеспечение аутентификации, целостности и конфиденциальности данных при обмене в недоверенной сети, такой как Интернет.

Архитектура протокола состоит из двух основных компонентов. Протокол рукопожатия (Handshake Protocol) отвечал за аутентификацию сторон и согласование криптографических алгоритмов и сессионных ключей. Протокол записи (Record Protocol) использовал установленные параметры для фрагментации, сжатия и шифрования передаваемых данных.

Несмотря на его историческую важность, протокол SSL 3.0 является полностью устаревшим и содержит критические уязвимости. Наиболее известная из них — POODLE (**Padding Oracle On Downgraded Legacy Encryption**), которая позволяет злоумышленнику расшифровать содержимое зашифрованного трафика. Использование SSL 3.0 категорически не рекомендуется. Все современные системы должны использовать его преемника — протокол TLS (Transport Layer Security).

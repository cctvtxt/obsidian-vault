[[HTTP Status Codes]]

##### 1* класс кодов (информационные сообщения). Это системный класс кодов, который только информирует о процессе передачи запроса. Такие ответы не являются ошибкой, хотя и могут отображаться в браузере как Error Code

100 Continue: сервер получил заголовки запроса и готов к получению тела. Позволяет клиенту не отправлять тело запроса, если заголовки неверны, что экономит трафик.

101 Switching Protocols: сервер подтверждает смену протокола по запросу клиента. Часто используется для перехода на WebSocket, обеспечивая постоянное двунаправленное соединение.

102 Processing: запрос принят и обрабатывается, но ответ еще не готов. Предотвращает отмену запроса клиентом из-за таймаута при длительных операциях.

103 Checkpoint: используется для возобновления прерванных запросов POST или PUT. Сервер сообщает, какая часть данных была принята, позволяя клиенту продолжить с этой точки.

105 Name Not Resolved: DNS-адрес сервера не может быть преобразован. Указывает на проблемы с DNS на стороне клиента или в сети, а не на ошибку сервера.

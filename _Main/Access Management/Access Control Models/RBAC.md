[[Access Control]]

Role-Based Access Control (RBAC), или Ролевое Управление Доступом, — это наиболее распространённая модель в корпоративных системах. Права доступа в ней назначаются не отдельным пользователям, а ролям, которые отражают должностные обязанности в организации (например, "Администратор", "Менеджер", "Пользователь").

Администратор сначала определяет набор разрешений для каждой роли, а затем присваивает пользователям одну или несколько ролей. Это значительно упрощает управление доступом в больших системах: вместо изменения прав для сотен пользователей достаточно изменить разрешения для одной роли. RBAC эффективно реализует принцип наименьших привилегий и хорошо масштабируется, однако может быть недостаточно гибким для правил, зависящих от контекста.

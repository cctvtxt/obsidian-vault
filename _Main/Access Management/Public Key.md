[[TLS-Based Auth]]

**Публичный (открытый) ключ (Public Key)** — это несекретный компонент криптографической пары ключей, который предназначен для свободного распространения. Он математически связан с `Private Key`, но получить приватный ключ из публичного невозможно. Его основное назначение — шифрование данных, которые сможет расшифровать только владелец приватного ключа, и проверка цифровых подписей, созданных приватным ключом.

---
## Генерация

Публичный ключ генерируется (извлекается) из `Private Key` в момент создания криптографической пары.

```bash
# Извлечение публичного ключа из уже существующего приватного
openssl rsa -pubout -in private_key.pem -out public_key.pem
```

## Хранение

Публичный ключ не является секретом и может храниться открыто.

### Сервер
*   **База данных**: Хранение публичных ключей клиентов для проверки их подписей.
*   **Файлы конфигурации**: Например, файл `~/.ssh/authorized_keys` на сервере хранит публичные ключи пользователей, которым разрешен SSH-доступ.

### Клиент
*   Клиент хранит публичные ключи серверов, которым он доверяет, чтобы шифровать для них данные или проверять их подлинность (например, файл `~/.ssh/known_hosts`).

## Передача

Публичный ключ можно безопасно передавать по любым, даже незащищенным каналам связи, так как его компрометация не несет прямого риска безопасности. Обычно он предоставляется всем сторонам, которые должны взаимодействовать с владельцем соответствующего приватного ключа.

## Использование
*   **Аутентификация**: Сервер использует публичный ключ клиента для проверки подписи, созданной приватным ключом клиента.
*   **Шифрование**: Клиент использует публичный ключ сервера для шифрования данных, которые затем могут быть расшифрованы только приватным ключом сервера. Это часто используется для безопасной передачи симметричного ключа при установке `TLS`-соединения.

[[OAuth-Based Auth]]

**Authorization Code** — это временный одноразовый код, который сервер авторизации выдаёт после успешной аутентификации пользователя и подтверждения прав доступа. Код передаётся клиенту через браузер в параметрах `redirect_uri`, но сам по себе не даёт доступа к защищённым ресурсам.

Его назначение — служить безопасным промежуточным звеном между пользователем и сервером. Обмен кода на `access_token` и `refresh_token` происходит по прямому сервер-серверному каналу (бэкенд → провайдер), что исключает передачу долгоживущих токенов через браузер и снижает риск их перехвата.

Используется в потоке **Authorization Code Flow**: код генерируется провайдером на этапе редиректа после согласия пользователя, затем клиент отправляет его на свой бэкенд, который обменивает код на токены через защищённый POST-запрос к эндпоинту `/token`.

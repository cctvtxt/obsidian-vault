[[OAuth-Based Auth]]

**Code Challenge** — это хеш-представление секрета `code_verifier`, вычисляемое по алгоритму `S256` (SHA-256 с base64url-кодированием) или `plain`. Параметр передаётся авторизационному серверу в начальном запросе вместе с `code_challenge_method`.

Введён в рамках расширения **PKCE (RFC 7636)** для защиты от атак перехвата авторизационного кода. Сервер авторизации сохраняет `code_challenge` и позже использует его для верификации: при обмене кода на токен клиент должен предъявить исходный `code_verifier`, хеш от которого должен совпасть с ранее переданным челленджем.

Используется на старте авторизационного потока: клиент генерирует `code_verifier`, вычисляет из него `code_challenge` и добавляет оба параметра (верифайер — локально, челлендж — в запрос) перед редиректом пользователя на провайдера.

[[Stateless Auth]]

Протокол описан в OASIS-стандарте "Assertions and Protocols for the OASIS Security Assertion Markup Language (SAML) V2.0". Интеграция SAML в OAuth 2.0 реализуется через RFC 7522, позволяя использовать SAML-утверждения как grant для получения OAuth-токенов.

До появления SAML технология единого входа была достижима, но базировалась на файлах cookies, которые по умолчанию жизнеспособны только в пределах одного домена


SAML фокусируется на федеративной аутентификации/авторизации между Identity Provider (IdP) и Service Provider (SP) через браузер: обмен SAML Request/Response в XML, подпись/шифрование, SSO без прямого API-вызова. В отличие от OAuth (grant types, token endpoint), здесь нет access/refresh-токенов — только утверждения для SSO

В SAML принцип единого входа обеспечивается за счет согласования процесса аутентификации с системой управления доступами. Когда пользователь запрашивает доступ к сайту, с помощью SAML передаются аутентификационные данные поставщику услуг, который теперь предоставляет доступ на вход. 

Используется только для аутентификации. Предоставляет утверждения о пользователе, содержащие информацию о его аутенификации и атрибутах. Использует XML для передачи утверждений о безопасности. 

Обычно используется для реализации SSO в корпоративных системах, принцип единого входа обеспечивается за счет согласования процесса аутентификации с системой управления доступом.

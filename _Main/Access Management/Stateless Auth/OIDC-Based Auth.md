[[Stateless Auth]]

**Open ID Connect** - это расширение **OAuth 2.0**, открытый стандарт, предоставляющей пользователю возможность создать единую учетную запись для аутентификации на множестве не связанных друг с другом интернет-ресурсов. **OAuth 2.0** не имеет стандарта передачи пользовательских данных. Протокол добавляет сведения о логине и профиле пользователя, который вошел в учетную запись, вводит **ID Token** — JWT-токен, содержащий утверждения (claims) о пользователе: sub (идентификатор), iss (эмитент), aud (аудитория), exp (срок действия), а также дополнительные данные вроде email, name или profile

Этот токен подтверждает, что пользователь аутентифицирован, **чего OAuth 2.0 не предоставляет сам по себе** — OAuth фокусируется только на доступе к ресурсам через access_token. Помимо этого, **Open ID Connect** позволяет реализовать Single Sign-On (SSO) и тем самым дает возможность входить в несколько приложений/несколько веб-сайтов с использованием единовременной аутентификации пользователя. 

Информацию о пользователе, вошедшем в систему (о Resource Owner'е), часто называют личными данными **identity**, а **Authorization Server** в контексте OIDC — поставщиком личных данных **IDP.**

## Плюсы Open ID Connect

- **Простота и совместимость**: Легко интегрируется с OAuth 2.0, использует JSON/JWT вместо XML (как SAML), упрощает SSO для веб-, мобильных и SPA-приложений
- **Stateless аутентификация и Federated-подход**: ID Token (JWT) верифицируется клиентом локально по публичному ключу из /jwks — нет нужды в базе состояний или звонках на introspection endpoint
## Минусы Open ID Connect

- **Ограниченные атрибуты**: Базовые claims (profile, email); для enterprise-атрибутов нужны custom claims или SAML — меньше гибкости
- **Зависимость от JWT**: протокол стандартизирует использование JSON Web Token (JWT, RFC 7519) как обязательный формат для **ID Token**, делая его основным механизмом передачи данных об аутентификации
- **Безопасные риски**: Implicit/Hybrid flows уязвимы к утечкам в браузере; SSO **увеличивает blast radius** при компрометации сессии IdP.​

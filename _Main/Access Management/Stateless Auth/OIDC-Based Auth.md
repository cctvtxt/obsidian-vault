[[Stateless Auth]] [[SSO]]

**Open ID Connect** - это расширение **OAuth 2.0**, открытый стандарт, предоставляющей пользователю возможность создать единую учетную запись для аутентификации на множестве не связанных друг с другом интернет-ресурсов. **OAuth 2.0** не имеет стандарта передачи пользовательских данных, а также соответствующих эндпоинтов. **Open ID Connect** решает эту проблему.

Протокол добавляет сведения о логине и профиле пользователя, который вошел в учетную запись, вводит **ID Token** — JWT-токен, содержащий утверждения (claims) о пользователе: sub (идентификатор), iss (эмитент), aud (аудитория), exp (срок действия), а также дополнительные данные вроде email, name или profile. Этот токен подтверждает, что пользователь аутентифицирован, **чего OAuth 2.0 не предоставляет сам по себе** — OAuth фокусируется только на доступе к ресурсам через access_token. Помимо этого, **Open ID Connect** позволяет реализовать Single Sign-On (SSO) и тем самым дает возможность входить в несколько приложений/несколько веб-сайтов с использованием единовременной аутентификации пользователя. 

Информацию о пользователе, вошедшем в систему (о Resource Owner'е), часто называют личными данными **identity**, а **Authorization Server** в контексте OIDC — поставщиком личных данных **IDP.**
_____
По сути расширение разработано для аутентификации пользователей. Добавляет идентификатор пользователя к токену доступа OAuth 2.0. OIDC вносит изменения в существующие OAuth endpoints и добавляет новые, плюс стандартизирует scopes и токены для аутентификации.

Его главный **плюс** это баланс между безопасностью OAuth 2.0 и простотой.

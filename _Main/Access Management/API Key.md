[[Key-Based Auth]]

**Ключ API (API Key)** — это уникальная строка символов, используемая для идентификации и аутентификации приложения (или пользователя) при выполнении запросов к API. Это простой механизм контроля доступа, который не требует сложного обмена токенами, как в OAuth.

---
## Генерация

### Сервер
Ключи всегда генерируются на стороне сервера. Обычно это происходит, когда разработчик регистрирует свое приложение в сервисе. Ключ должен быть длинной, криптографически случайной строкой.

```javascript
// Пример генерации на Node.js
const apiKey = crypto.randomBytes(32).toString('hex');
```

### Клиент
Клиент (приложение-потребитель API) не генерирует, а только получает и использует ключ, выданный сервером.

## Хранение

### Сервер
*   **Никогда не храните API ключи в открытом виде.** В базе данных следует хранить только **хеш** от ключа (например, `SHA-256`).
*   При получении запроса сервер хеширует входящий ключ и сравнивает его с хешем в базе данных. Это защищает от захвата всех ключей при утечке базы.
    ```sql
    SELECT user_id FROM api_keys WHERE key_hash = SHA256('provided_api_key');
    ```

### Клиент (приложение-потребитель)
*   **На сервере клиента**: Ключ API должен рассматриваться как пароль. Его следует хранить в `переменных окружения`, `.env` файле или через систему управления секретами.
*   **В браузерном коде (JavaScript)**: **НИКОГДА** не храните и не вставляйте API ключ в код, доступный клиенту. Это приведет к его неминуемой краже. Для браузерных приложений используйте `Access Token` или проксируйте запросы через свой бэкенд, который уже будет добавлять API ключ.

## Передача

Передача ключа **обязательно** должна происходить по зашифрованному каналу (`HTTPS`).

Существует несколько распространенных способов:
1.  **Пользовательский HTTP-заголовок (рекомендуется)**:
    ```http
    GET /api/data
    Host: example.com
    X-API-Key: <your_api_key>
    ```
2.  **Заголовок `Authorization`**:
    ```http
    GET /api/data
    Host: example.com
    Authorization: ApiKey <your_api_key>
    ```
3.  **Параметр запроса (менее безопасно)**:
    ```http
    GET /api/data?apiKey=<your_api_key>
    ```
    Этот способ не рекомендуется, так как URL-адреса, включая параметры, часто логируются серверами и могут попасть в историю браузера.

## Использование
Сервер использует ключ для:
*   **Аутентификации**: Кто делает запрос?
*   **Авторизации**: Что этому приложению разрешено делать?
*   **Управления трафиком (Rate Limiting)**: Ограничение количества запросов в единицу времени для каждого ключа.

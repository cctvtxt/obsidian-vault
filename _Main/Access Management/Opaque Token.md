[[OAuth-Based Auth]]

Непрозрачные токены — это тип аутентификационных токенов, которые сами по себе не несут никакой смысловой нагрузки. Обычно это случайные наборы символов, которые используются в качестве ссылок на фактические данные аутентификации и пользовательские данные на сервере.

Opaque-токены валидируются на сервере путём обращения к внешнему authorization server (AS) через introspection endpoint (RFC 7662), без необходимости хранить их локально на resource server. Это **stateful** подход на стороне AS, но **stateless** для сервера

**Introspection** — это POST-запрос на `/oauth/introspect` (или аналогичный путь) AS с параметрами `token` и `token_type_hint=access_token` в body 

- **Нечитаемые**: они не содержат данных. Их ценность заключается в уникальности и непредсказуемости
- **Ссылочные токены**: они ссылаются на данные, хранящиеся в другом месте, часто на централизованном сервере или в базе данных.
- **Безопасные**: поскольку непрозрачные токены не содержат данных, даже в случае их перехвата они сами по себе не раскрывают никакой информации
- **Легковесные**: небольшие, поскольку не содержат полезных данных
- **Основной недостаток токенов** состоит в том, что для получения, обновления и проверки токенов необходима связь с централизованным хранилищем

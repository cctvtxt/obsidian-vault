### Концепции в запросах GraphQL

**1. Поля (Fields)**

Поля — это **основные единицы данных**, которые клиент запрашивает у сервера. Они представляют атрибуты или свойства объекта (например, ID, имя, email). Поля являются строительными блоками GraphQL-запросов и позволяют точно указать, какие данные нужно получить.

```text
{ user { name email } }
```

Сервер вернёт:

```text
{ "data": { "user": { "name": "Иван Петров", "email": "ivanpetrov@example.com" } } }
```

**2. Аргументы (Arguments)**

Аргументы **уточняют запрос**, позволяя фильтровать, сортировать или ограничивать результаты. Передаются в круглых скобках после имени поля в формате `имя: значение`.

```text
{ books(limit: 3, genre: "фантастика") { title author { name } } }
```

Здесь `limit` и `genre` — аргументы, которые фильтруют книги по жанру и ограничивают количество результатов.

**3. Псевдонимы (Aliases)**

Псевдонимы позволяют **избежать конфликтов имён** при запросе нескольких одинаковых полей и улучшают читаемость ответов. Псевдоним указывается перед полем или запросом, после него ставится двоеточие, затем — оригинальное имя поля или запроса.

```text
{ firstBook: book(id: "1") { name: title year: publishedYear } secondBook: book(id: "2") { name: title year: publishedYear } }
```

Ответ сервера:

```text
{ "data": { "firstBook": { "name": "1984", "year": 1949 }, "secondBook": { "name": "О дивный новый мир", "year": 1932 } } }
```

**4. Фрагменты (Fragments)**

Фрагменты позволяют **переиспользовать повторяющиеся части запросов**, группируя их в блоки. Это сокращает код и упрощает поддержку.

**5. Переменные (Variables)**

Переменные **повышают гибкость запросов**, позволяя использовать динамические параметры вместо фиксированных значений. Объявляются в начале запроса с символом `$`, за которым следует имя и тип переменной.

**6. Директивы (Directives)**

Директивы **управляют поведением запроса**, позволяя гибко включать или исключать поля в зависимости от условий.

**Основные директивы:**

- `@include(if: Boolean)` — включает поле в результат, если условие истинно;
- `@skip(if: Boolean)` — исключает поле, если условие истинно;
- `@deprecated(reason: String)` — помечает поле как устаревшее;
- `@specifiedBy` — указывает URL спецификации для пользовательского скалярного типа.

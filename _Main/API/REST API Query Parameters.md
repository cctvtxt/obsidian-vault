Параметры `limit`, `offset`, `page`, `sort`, `filter` — это **query-параметры** (параметры запроса), которые используются в API (например, REST API) и SQL-запросах для управления данными: фильтрации, сортировки и пагинации (разбиения на страницы). Они позволяют клиентским приложениям уточнять запросы к серверу .

**1. `limit`** — задаёт **количество записей**, которые нужно вернуть в ответе. Используется вместе с `offset` для реализации пагинации.  
**Пример:** `GET /api/products?limit=10` — вернёт 10 записей.  
В SQL: `SELECT * FROM table LIMIT 10`.

**2. `offset`** — указывает, **сколько записей пропустить** перед началом возврата данных. Работает в паре с `limit`.  
**Пример:** `GET /api/products?offset=20&limit=10` — пропустит первые 20 записей, вернёт следующие 10.  
В SQL: `SELECT * FROM table LIMIT 10 OFFSET 20`.

**3. `page`** — номер страницы при пагинации (альтернатива `offset`). Используется в некоторых API для упрощения навигации.  
**Пример:** `GET /api/products?page=3&limit=10` — вернёт 3-ю страницу (записи 21–30).  
Часто преобразуется сервером в `offset` (например, `offset = (page - 1) * limit`).

**4. `sort`** — задаёт **порядок сортировки** записей (по возрастанию/убыванию).  
**Пример:** `GET /api/products?sort=price&order=desc` — отсортирует товары по цене в порядке убывания.  
В SQL: `SELECT * FROM table ORDER BY price DESC`.

**5. `filter`** — позволяет **фильтровать данные** по определённым критериям.  
**Пример:** `GET /api/products?filter=category:electronics&filter=price_lt:1000` — вернёт электронику дешевле 1000 рублей.  
В SQL: `SELECT * FROM table WHERE category = 'electronics' AND price < 1000`.

### Прописываются ли они в заголовках запроса?

**Нет, эти параметры не прописываются в заголовках HTTP-запроса (Headers).** Они передаются **в URL-адресе запроса** как **query-параметры** (часть строки запроса после знака `?`), разделённые символом `&`.

**Формат URL с query-параметрами:**  
`https://api.example.com/resource?param1=value1&param2=value2&...`

**Пример полного URL с пагинацией и фильтрацией:**  
`https://api.example.com/products?page=2&limit=10&sort=name&filter=category:books`

**Где ещё могут передаваться:**

- В теле POST-запроса (редко, зависит от спецификации API).
- Как параметры в методах фреймворков (например, `@RequestParam` в Spring Boot).
- В SQL-запросах напрямую (как части синтаксиса `LIMIT`, `OFFSET`, `WHERE`, `ORDER BY`).

**Вывод:** query-параметры (`limit`, `offset` и др.) — это способ передачи инструкций серверу о том, как обработать данные, и они указываются **в строке запроса URL**, а не в HTTP-заголовках.

[[API]]

**Контракт API** — это соглашение между поставщиком API и его пользователями, в котором описываются предоставляемые методы API, поддерживаемые форматы и структуры данных, ожидаемые входные данные, возможные ответы, коды ошибок и другие важные инструкции.

Подход "Contract First" подразумевает, что проектирование и утверждение контракта происходит до написания основной логики приложения. Контракт становится единым источником правды как для серверной, так и для клиентской команды, позволяя вести разработку параллельно.

В экосистеме Node.js для описания контрактов чаще всего используется спецификация OpenAPI (ранее Swagger). Она позволяет формально описать API в формате YAML или JSON. На основе этого описания можно генерировать документацию, клиентские SDK и даже каркас серверного кода.

Пример простого контракта для получения пользователя по ID:

```yaml
openapi: 3.0.0
info:
  title: User API
  version: 1.0.0
paths:
  /users/{id}:
    get:
      summary: Get user by ID
      parameters:
        - name: id
          in: path
          required: true
          schema:
            type: string
      responses:
        '200':
          description: A single user
          content:
            application/json:
              schema:
                type: object
                properties:
                  id:
                    type: string
                  name:
                    type: string
        '404':
          description: User not found
```

Использование контрактов обеспечивает предсказуемость, упрощает тестирование и интеграцию, а также служит надежной документацией, которая всегда остается актуальной.

Помимо формальных API контрактов, в программировании существуют и неявные контракты на уровне кода. Это соглашения о том, как функции должны себя вести.

Одним из старейших контрактов в Node.js является "Error-First Callback". По этому соглашению, функция обратного вызова (callback), передаваемая в асинхронную операцию, всегда принимает ошибку первым аргументом. Если ошибки нет, этот аргумент равен `null`.

```javascript
const fs = require('fs');

fs.readFile('/path/to/file', (err, data) => {
  // Контракт: первым аргументом будет ошибка или null
  if (err) {
    console.error('Ошибка чтения файла:', err);
    return;
  }
  // Второй и последующие аргументы содержат результат
  console.log(data);
});
```

С развитием языка появились более совершенные контракты для асинхронных операций, такие как Промисы (Promises). Промис — это объект, представляющий будущее завершение (или провал) асинхронной операции. Он является контрактом, который обещает вернуть результат или ошибку.

```javascript
// Контракт: функция возвращает промис
function readFilePromise(path) {
  return new Promise((resolve, reject) => {
    fs.readFile(path, (err, data) => {
      if (err) {
        // Нарушение контракта (провал)
        reject(err);
        return;
      }
      // Успешное выполнение контракта
      resolve(data);
    });
  });
}

readFilePromise('/path/to/file')
  .then(data => console.log(data))
  .catch(err => console.error('Ошибка:', err));
```

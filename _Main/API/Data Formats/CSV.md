### CSV (.csv)

Comma-Separated Values – текстовый формат для представления табличных данных.

#### Преимущества

- Простой и понятный формат
- Человекочитаемый
- Поддерживается практически везде
- Легко создавать и редактировать

#### Недостатки

- Неэффективное использование места
- Медленное чтение больших файлов
- Проблемы с типами данных
- Сложности с обработкой специальных символов

Пример Создан в 1980-х как простой текстовый формат для табличных данных (Comma-Separated Values). **Характеристика: Табличный, разделитель-based (запятая по умолчанию), минималистичный для списков строк — "Comma-Separated Values" (CSV).** Цель — обмен данными между программами без бинарных форматов, стандартизирован RFC 4180 в 2005. CSV использует строки с разделителями (,), кавычки для полей, поддерживает базовые типы: строки, числа, даты (без типизации). Комментарии нестандартны, escape с кавычками "".

```csv
dependency,version,features
serde,1.0,"[""derive""]"
toml,0.8, build_version,0.1.0
build_date,2026-03-04T20:39:00Z
build_authors,"dev@example.com"
```

Формат эволюционировал от Excel-табов к RFC-стандарту. Альтернатива TSV (tab), JSON (для таблиц проще), XML (минимализм). Часто сравнивают с базой данных dump для импорта.

## Область применения

CSV повсеместен для данных: экспорт Excel/Sheets, базы данных (SQL dump), логи, ETL (Pandas, Apache Spark), API-ответы таблиц. Идеален для простого обмена табличными данными без сложной структуры.

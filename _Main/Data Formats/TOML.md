Создан Томом Престон-Вернером (сооснователем GitHub) в 2013 году как альтернатива YAML для проекта Hugo. Цель — очевидность и простота, с открытой спецификацией, поддерживаемой сообществом. Версии эволюционировали: v0.1.0 в 2013, v1.0.0 в 2017. TOML организует данные в таблицы (с помощью `[table]`), поддерживает базовые типы: строки, числа, булевы значения, даты, массивы и вложенные таблицы. Комментарии начинаются с `#`, строки могут быть многострочными, синтаксис минималистичен без лишних скобок.​

```toml
[dependencies]
serde = { version = "1.0", features = ["derive"] }
toml = "0.8"

[build]
version = "0.1.0"
date = 2026-03-04T20:39:00Z
authors = ["dev@example.com"]
```

Формат разработан как улучшенный INI (разделы как в INI, но мощнее). Альтернатива JSON (добавляет комментарии, типы), YAML (проще, без неоднозначностей). Часто сравнивают с XML для конфигов.

## Преимущества

- Высокая читаемость для человека, особенно с вложенностью.
- Поддержка комментариев и строгой типизации (даты, время).
- Меньше ошибок благодаря простому синтаксису и гибкости (массивы, таблицы).
- Широкая межъязыковая поддержка (Python, Rust, Go и др.).

## Недостатки

- Меньшая распространенность по сравнению с JSON/YAML, что ограничивает библиотеки.
- Кривая обучения для специфического синтаксиса таблиц.
- Не идеален для сложного обмена данными или глубоких структур без схем.
- Таблицы заполняются целиком, что усложняет слияние файлов.

## Область применения

TOML популярен для конфигураций: Cargo (Rust), Python Poetry/Pipenv, Hugo, tmux. Идеален для проектов, где важна читаемость настроек без сложной иерархии.

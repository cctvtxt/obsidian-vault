Пример HTML создан Тимом Бернерс-Ли в 1991 году (CERN) как язык разметки гипертекста для WWW. **Характеристика: Теговый (tag-based), декларативный, структурированный для веб-документов — "HyperText Markup Language" (HTML).** Цель — представление контента с семантикой, гиперссылками, мультимедиа. Версии эволюционировали: HTML 1.0 (1991), 4.01 (1999), 5 (2014, живой стандарт WHATWG). HTML использует парные теги `<tag>content</tag>`, атрибуты, DOCTYPE, поддерживает текст, изображения, формы, скрипты через JS. Комментарии `<!-- -->`.

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <title>Проект</title>
  </head>
  <body>
    <header>
      <h1>Dependencies</h1>
    </header>

    <section class="dependencies">
      <ul>
        <li>
          <strong>serde</strong>: version 1.0,
          <em>features: ["derive"]</em>
        </li>
        <li>toml: 0.8</li>
      </ul>
    </section>

    <footer class="build-info">
      <p>
        Version: 0.1.0 | Date: 2026-03-04T20:39:00Z | Author: dev@example.com
      </p>
    </footer>
  </body>
</html>
```

Формат позиционируется как основа веба (W3C/WHATWG), альтернатива XML (проще, с дефолтными стилями), Markdown (богаче семантика). Эволюционировал от простых страниц к SPA-приложениям.

## Структура документа

HTML строится как дерево DOM с обязательной структурой:

**Обязательные элементы:**

- `<!DOCTYPE html>` — объявляет HTML5.
- `<html>` — корень, содержит `lang`.
- `<head>` — метаданные (title, meta, link CSS, script).
- `<body>` — видимый контент.

**Семантическая структура (HTML5):**

```text
html
├── head (метаданные)
├── body
    ├── header (шапка)
    ├── nav (навигация)
    ├── main (основной контент)
    │   ├── article (статья)
    │   ├── section (раздел)
    │   └── aside (боковая панель)
    ├── footer (подвал)

```

**Иерархия тегов:**

- Блочные: `<div>`, `<section>`, `<article>` — контейнеры.
- Встроенные: `<span>`, `<em>`, `<strong>` — внутри строк.
- Семантические: `<header>`, `<nav>`, `<footer>`, `<main>`.

## Преимущества

Универсальность веба — браузеры, поисковики, скринридеры.  
Семантика для доступности (ARIA, screen readers).  
Экосистема: CSS, JS, фреймворки (React/Vue).  
Живой стандарт — perpetual beta без ломки.  
Поддержка мультимедиа (video, canvas, webgl).

## Недостатки

Вербозность для данных (лучше JSON).  
Парсинг lenient — допускает ошибки.  
Зависимость от CSS/JS для стиля/логики.  
Устаревшие атрибуты (align, bgcolor).  
Сложность валидации больших документов.

## Область применения

**Веб-разработка:** SPA (React/Angular), статические сайты (Hugo/Jekyll), CMS (WordPress).  
**Документация:** GitHub README, MDX, Sphinx.  
**Enterprise:** отчеты (JSP), legacy веб-приложения.  
**Мобильное:** Cordova/PhoneGap hybrid apps.

Идеален для структурированного контента с гиперссылками и семантикой.

[[Browser]]

Внутренняя обработка TLS браузером — это задача его сетевого движка, который активируется при обнаружении схемы `https://` в URL. Этот компонент отвечает за инициирование и управление всем процессом установления безопасного соединения, прозрачно для движка рендеринга, который работает с контентом (HTML, CSS).

Процесс рукопожатия начинается, когда сетевой стек браузера формирует и отправляет пакет `ClientHello`, предлагая поддерживаемые им шифронаборы. Ключевой этап для браузера наступает при получении ответа от сервера. Внутренний модуль безопасности браузера выполняет строгую проверку серверного сертификата: он сверяет его цифровую подпись с набором корневых сертификатов из собственного хранилища доверенных центров сертификации (Trust Store). Также проверяется соответствие доменного имени в сертификате с URL, срок действия и статус отзыва через протоколы OCSP или CRL. Пользовательский интерфейс браузера, например, иконка замка в адресной строке, напрямую отражает результат этой проверки.

После успешного рукопожатия и генерации сессионного ключа сетевой движок браузера переходит в режим шифрования. Когда движок рендеринга запрашивает ресурс (например, `GET /style.css`), запрос не отправляется напрямую в сеть. Вместо этого он передается внутреннему криптографическому модулю (например, BoringSSL, NSS), который инкапсулирует HTTP-сообщение в TLS-записи, шифрует их и затем отправляет по TCP-соединению. При получении ответа происходит обратный процесс: сетевой стек читает TLS-записи, криптографический модуль их расшифровывает и передает чистое HTTP-сообщение движку рендеринга для дальнейшей обработки.

### Как организовать связь между Docker-контейнерами

Одним из ключевых преимуществ Docker является возможность запускать приложения в изолированных средах — контейнерах. Однако для работы многокомпонентных приложений (например, состоящих из веб-сервера и базы данных) контейнеры должны взаимодействовать друг с другом. Для этого Docker использует **виртуальные сети** — гибкую систему, которая позволяет контролировать соединения, безопасность и маршрутизацию.

#### Как работает связь между контейнерами

Каждый контейнер в Docker имеет собственный сетевой стек, изолированный от хост-машины и других контейнеров. Чтобы контейнеры могли общаться, Docker создаёт виртуальные сети, в которых они могут обмениваться данными. При этом связь может быть организована разными способами в зависимости от задач и архитектуры приложения.

#### Основные типы сетей в Docker

1. **Bridge (по умолчанию)** — наиболее распространённый способ организации связи между контейнерами на одной хост-машине. При установке Docker автоматически создаётся сеть `bridge`, к которой подключаются контейнеры, если сеть не указана явно.
   - **Как работает:** Docker назначает каждому контейнеру виртуальный IP-адрес и позволяет им общаться по этим адресам.
   - **Особенность:** в пользовательских bridge-сетях Docker использует встроенный DNS-сервер — контейнеры могут обращаться друг к другу не только по IP, но и по именам. Например, если у вас есть контейнер с базой данных под именем `db`, другой контейнер может подключиться к нему просто по имени `db:5432` (где `5432` — порт базы данных).

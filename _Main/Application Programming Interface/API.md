**API** - это контракт, который предоставляет программа. То есть присутствует определенная функция/скрипт, где прописаны способы/условия взаимодействия между клиентом и сервером. **API включает в себя:**

1. _Операция_
2. _Данные на входе_
3. _Данные на выходе_

**API** (**Application Programming Interface**) — это набор правил (интерфейс) и протоколов, которым соглашаются следовать две отдельные системы или программы — одна на стороне публикации, а другая на стороне доступа, позволяющих приложениям взаимодействовать друг с другом
_____
В то же время **протокол** — это **набор правил и соглашений**, описывающих, как устройства или программы обмениваются данными: структура пакетов, порядок сообщений, способы установления и завершения соединений. Он регулирует:

1. установление соединения и приветствие сторон;
2. согласование параметров передачи (формат, скорость, кодировка и т. п.);
3. структуру и формат передаваемых сообщений;
4. механизмы подтверждения приёма и контроля целостности данных;
5. обработку ошибок и восстановление после сбоев;
6. порядок завершения сеанса связи;
7. адресацию и маршрутизацию (для сетевых протоколов).
______
**Протокол** отвечает на вопрос «как физически и логически передавать данные» (уровень транспорта и формата сообщений). **API** отвечает на вопрос «что именно можно попросить у сервиса и в каком виде» (уровень бизнес-операций). Один и тот же API можно реализовать поверх разных протоколов (например, SOAP может работать поверх HTTP, SMTP и т.д.), а один протокол может использоваться разными API (REST, GraphQL и т.п.).
__________
**Web API** — это специально созданный программный интерфейс, используемый для веб-серверов или веб-браузеров, который ограничен на стороне клиента. Другими словами, Web API не включает детали реализации сервера или браузера, если они не являются общедоступными.

API обычно делится на четыре типа, которые могут отличаться по некоторым признакам:

- **Public API** — открытый тип API, основной особенностью которого является доступность для любого человека вне бизнес-процесса.
- **Partner API** — второй тип, который определяется ограниченным доступом, обычно предоставляемым внешним разработчикам.
- **Internal API** — тип API, часто называемый частным, созданный с целью предоставления доступа только членам проектной команды или бизнеса.
- **Composite API** — представляет собой комбинацию нескольких API с целью построения последовательности операций, которые могут быть как связанными, так и независимыми.

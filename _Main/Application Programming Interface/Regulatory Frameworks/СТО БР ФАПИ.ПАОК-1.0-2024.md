[[API Compliance]]

СТО БР ФАПИ.ПАОК-1.0-2024 — это стандарт Банка России, описывающий поведенческий анализ и обнаружение компьютерных атак при использовании открытых API. Он дополняет другие стандарты по безопасности, фокусируясь на динамическом анализе и выявлении аномалий в реальном времени.

Стандарт решает задачу обнаружения и предотвращения сложных и нетипичных атак на финансовые API, которые не могут быть выявлены статическими правилами безопасности (например, атаки на бизнес-логику, мошенничество с использованием ботов, сложные схемы фрода).

Документ вводит требования к сбору и анализу поведенческих данных: параметров запросов, последовательности вызовов API, временных характеристик и других. Он описывает необходимость построения моделей «нормального» поведения пользователей и систем и выявления отклонений от этих моделей.

# Вводимые ограничения

API и связанная с ним инфраструктура должны обеспечивать сбор **расширенных данных для поведенческого анализа**. Это не только стандартные логи, но и специфические метрики: частота и последовательность вызовов эндпоинтов, объем передаваемых данных, геолокация, цифровые отпечатки устройств (`fingerprinting`).

Требуется внедрение специализированных систем **поведенческого анализа (User and Entity Behavior Analytics, UEBA)** или систем класса **Anti-Fraud**. Эти системы должны в реальном времени анализировать поток событий от API, сопоставлять их с моделями нормального поведения и генерировать алерты при выявлении аномалий.

Система должна иметь механизмы **автоматического реагирования** на выявленные угрозы. Это может включать блокировку подозрительной сессии или IP-адреса, требование дополнительной аутентификации (step-up authentication) или временное ограничение функциональности для подозрительного пользователя.

# Практический путь внедрения

Внедрение поведенческого анализа является передовым рубежом в обеспечении безопасности API. Этот процесс требует не только закупки или разработки сложных аналитических систем, но и привлечения специалистов по Data Science для построения и обучения моделей.

Разработка API усложняется необходимостью его глубокой интеграции с системами анализа. API должен предоставлять аналитической платформе богатый контекст о каждой операции. Основная сложность заключается в минимизации ложноположительных срабатываний (false positives), чтобы не блокировать легитимных пользователей. Несмотря на сложность, внедрение поведенческого анализа позволяет перейти от реактивной к проактивной модели безопасности, обнаруживая атаки на самых ранних стадиях.

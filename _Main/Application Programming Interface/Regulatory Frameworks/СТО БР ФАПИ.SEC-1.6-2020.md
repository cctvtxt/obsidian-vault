[[API Compliance]]

СТО БР ФАПИ.SEC-1.6-2020 — это стандарт Банка России, посвященный безопасности финансовых (банковских) операций, совершаемых с использованием открытых программных интерфейсов (API). Он устанавливает конкретные технические требования к безопасности API в финансовом секторе РФ.

Стандарт решает задачу защиты от атак, направленных на компрометацию API финансовых организаций, и обеспечение целостности, конфиденциальности и доступности данных при межсистемном взаимодействии.

Документ детализирует требования к аутентификации и авторизации клиентов и приложений, криптографической защите передаваемых данных, ведению журналов аудита, а также к защите от типовых векторов атак на API, таких как инъекции, межсайтовый скриптинг (XSS) и подделка межсайтовых запросов (CSRF).

# Вводимые ограничения

Стандарт требует применения **строгой взаимной аутентификации (mTLS)** между участниками обмена. Как клиент, так и сервер должны использовать цифровые сертификаты, выданные доверенным удостоверяющим центром, для подтверждения своей подлинности на транспортном уровне.

Все сообщения, передаваемые через API, должны быть подписаны **усиленной квалифицированной электронной подписью (УКЭП)**. Это обеспечивает юридическую значимость, целостность и неотказуемость операций, совершаемых через API.

Требуется детальное **логирование** всех операций, включая информацию об участниках, времени, содержании запросов и ответов, а также результатах проверки электронных подписей. Журналы должны быть защищены от модификации и храниться в течение установленного срока.

# Практический путь внедрения

Внедрение стандарта является обязательным для банков, предоставляющих доступ к своим сервисам через открытые API в рамках концепции Open Banking в РФ. Это один из наиболее технически сложных стандартов, требующий глубоких знаний в области криптографии и информационной безопасности.

Разработка API усложняется необходимостью интеграции с криптографическими провайдерами (СКЗИ), удостоверяющими центрами и системами управления сертификатами. Процессы CI/CD должны быть адаптированы для работы с электронной подписью и тестирования криптографических механизмов. Несмотря на высокую сложность, соответствие стандарту является ключом к построению безопасной и доверенной экосистемы открытых финансов.

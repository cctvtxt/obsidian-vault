[[API]]

Способность API соответствовать регуляторным, отраслевым и внутренним стандартам.

GDPR
CCPA
HIPPA
PCI DSS
PII


|Характеристика|Описание|Метрики / Критерии|Примеры реализации|
|---|---|---|---|
|**Регуляторное соответствие**|Соответствие законам и стандартам|✓ Аудиты пройдены, ✓ Нет штрафов|GDPR, HIPAA, PCI-DSS, SOX, 152-ФЗ|
|**Стандарты данных**|Форматы, кодировки, схемы|✓ Валидация по схеме, ✓ Консистентность|JSON Schema, OpenAPI 3.1, ISO 8601, UTF-8|
|**Аудит и трассируемость**|Фиксация всех значимых действий|✓ 100% критических операций залогировано, ✓ Retention ≥ N дней|Audit logs, immutable storage, SIEM-интеграция|
|**Управление согласием**|Контроль обработки персональных данных|✓ Явное согласие, ✓ Право на удаление|Consent flags, `DELETE /users/{id}`, data anonymization|
|**Гео-ограничения**|Соблюдение требований к локализации данных|✓ Данные не покидают регион, ✓ Маршрутизация по юрисдикции|Data residency policies, region-aware routing|
|**Лицензирование и квоты**|Контроль использования по тарифам|✓ Точный учёт запросов, ✓ Блокировка при превышении|Usage meters, quota headers, billing API|

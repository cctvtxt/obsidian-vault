# Obsidian Flashcards Builder Skill

## Триггеры активации

Скилл активируется при наличии:

- либо упоминания **flashcards** ИЛИ **spaced repetition** (регистр не важен)
- либо контекста: создание/добавление/редактирование/исправление тестов или карточек

## Структура проекта

## Именование файлов

### Формат

`[тема]-[подтема]-[подтема]-[подтема].md` — всегда lowercase, слова через дефис

Количество подтем не больше трех.

### Правило выбора имени

1. Если пользователь указал имя → использовать как есть (проверить lowercase + дефисы)
2. Если не указан → проанализировать существующие файлы в `/Tests`:
   - Найти 3 ближайших по тематике
   - Предложить варианты в формате:
     ```
     Предлагаю имена файлов на основе существующих:
     1. js-core-closures.md (как в js-promises.md)
     2. javascript-closures.md (как в javascript-async.md)
     3. Укажите свой вариант
     ```

## Работа с существующими файлами

### При добавлении вопросов

1. Найти последнюю карточку в файле
2. Добавить одну пустую строку после неё
3. Вставить новую карточку без пустых строк внутри
4. **Удалить ВСЕ метаданные SR**: `<!--SR:!2026-04-07,3,250-->` и подобные
5. Не трогать существующие вопросы (даже если кажутся неправильными)

## Структура файла (порядок строгий)

```
# [Название теста]

#flashcards/[тема]/[подтема]

[ВОПРОС 1]
?
[ОТВЕТ 1]

[ВОПРОС 2]
?
[ОТВЕТ 2]
```

---

## Формат [ВОПРОС] ? [ОТВЕТ]

### Базовая структура (ОБЯЗАТЕЛЬНО)

```
Текст вопроса, максимально конкретный и однозначный?
- вариант ответа 1 (опционально)
- вариант ответа 2 (опционально)
?
Краткий ответ — 1-3 предложения максимум
Дополнительное пояснение только если критично для понимания
Источник: [[название-заметки]] или путь (только если указан пользователем)
```

### Правила разделителей

**ДОПУСТИМО:**

- Одна пустая строка МЕЖДУ полными парами вопрос-ответ
- Перенос строки (`\n`) внутри вопроса для вариантов ответа

❌ **ЗАПРЕЩЕНО:**

- Пустая строка между вопросом и `?`
- Пустая строка между `?` и ответом
- Пустая строка внутри ответа (если это не разделение на "краткий/развёрнутый")
- Более одной пустой строки между карточками

## Выбор хештега

### Иерархия

`#flashcards/[категория]/[подкатегория]`

### Правила

- Категория = общая область (js, medicine, history)
- Подкатегория = конкретная тема (core, anatomy, ww2)
- Всегда kebab-case, без пробелов
- Если не уверен — предложи 2-3 варианта в ответе

---

### При исправлении

- Уточни у пользователя: "Исправить существующий вопрос или добавить новый?"
- Если исправляешь — сохранить позицию, заменить только текст

## Специальные форматы

### Вопросы с кодом

Использовать обратные кавычки:

````
Что выведет этот код?
```javascript
const x = 5;
console.log(x++);
````

?
5 (постфиксный инкремент возвращает старое значение)

```

### Формулы (MathJax)
```

Какая формула Эйлера?
?
$$e^{i\pi} + 1 = 0$$

```

### Изображения
```

Что изображено на схеме?
?
![[Pasted image 20240411.png]]
Структура нейрона (сома, дендриты, аксон)


```

│   НЕТ
│   ↓

```

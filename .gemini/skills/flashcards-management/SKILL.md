# Obsidian Flashcards Builder Skill

## Триггеры активации

Скилл активируется при:

- контексте работы с тестами/карточками в файлах: создание/добавление/редактирование/исправление тестов или карточек (в данной базе знаний, если нет специального другого контекста, под тестами и карточками в запросах с наибольшей вероятностью продразумеваются flashcards-карточки);
- при контексте работы с файлами для **flashcards** ИЛИ **spaced repetition** (регистр не важен).

## Создание flashcards

### Структура проекта

- MD-файлы с тестами находятся в директории /Tests.
- Формат имен создаваемых файлов всегда lowercase, слова на английском языке через дефис: `[тема]-[подтема]-[подтема]-[подтема].md`, количество подтем: от 1 до 3.
- Если имя файла указано в запросе, использовать как есть (проверить lowercase + дефисы).
- Усли имя в запросе не указано, основывай выбор на тематике теста, плюс перед именованием проанализируй наименвоания тестов в папке /Tests, чтобы увидеть общий стиль именования файлов и приблизиться к нему.
- Имя файла всегда соответствует хештегу внутри файла без первоначального слова `flashcards`: если в запросе указан хештег, но не имя, формируй имя по хештегу. `:#flashcards/auth/roles` это `auth-roles.md`

### Структура файла

1. Файл всегда в заголовке содержит наименование направленности конкретного теста.
2. После переноса строки указывается хештег: `#flashcards/[тема]/[подтема]`, как и наименование файла, может иметь не более 3 подтем.
3. После пропуска строки: пара вопрос-ответ.
4. Пропуск строки для разделения созданной карточки от последующей.

```
# [Название теста]

#flashcards/[тема]/[подтема]

[ВОПРОС 1]
?
[ОТВЕТ 1]

[ВОПРОС 2]
?
[ОТВЕТ 2]
```

### Структура карточек

- на отдельную карточку выносятся следующие данные: [ВОПРОС], перенос строки, знак ?, перенос строки [ОТВЕТ].
- Язык повествования всегда русский, понятия и термины пиши на английском.
- Проверяй, чтобы при генерации flashcars вопрос действительно всегда содержал весь контекст, передающий суть вопроса, но без ненужной воды.
- Внутри текста вопроса и ответа допускается только переход на новую строку, не пропуск строки, текст всегда идет последовательно.
- Пустая строка используется для разделения текста разных карточек.
- Запрещено появление более одной пустой строки между карточками.

```
Текст вопроса, максимально конкретный и однозначный?
- вариант ответа 1 (опционально)
- вариант ответа 2 (опционально)
?
Краткий ответ — 1-3 предложения максимум
Дополнительное пояснение только если критично для понимания
Источник: [[название-заметки]] или путь (только если указан пользователем)
```

### Редактирвоание карточек

- Если вопрос после создания был решен, то в конце вопроса перед разделяющим пропуском строки сохраняются метаданные в виде `<!--SR:!2026-04-07,3,250-->`, их необходимо удалить для карточек, которые были по запросу отредактирвоаны.
- при добавлении новых карточек не ломай структуру файла и не редактируй другие карточки, если это требование не было предъявлено в запросе.

## Отчёт о выполнении

После генерации (или редактирования) необходимо всегда выводить отчет в форме:

```markdown
**Создано flashcards:**

- Файл: `Tests/[имя-файла].md`
- Хештег: `#flashcards/[тема]/[подтема]`
- Количество вопросов: [N]
- Особенности: [код/формулы/изображения, если есть]
```

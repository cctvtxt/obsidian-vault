# Obsidian Flashcards Builder Skill (Spaced Repetition)

Создавай новые файлы с flashcards или дорабатывай существующие для работы с ними через плагин **Obsidian Spaced Repetition**.  

## Структура создаваемых Flash Cards

Синтаксис создания заметок должен быть следующим: хештег #flashcards в начале файла, точный и конкретный вопрос, знак вопроса, ответ, пропуск строки, следующий вопрос.

Не допускай появления пустых строк внутри пары вопрос-ответ - пропуск строки только для разделения вопросов. Внутри вопросов используй только перенос на следующую строку. Пример синтаксиса:

```
## Тест по теме Example

#flashcards/тема/подтема 

Текст вопроса, максимально однозначный понятный
- если необходимо, вариант ответа
- вариант ответа 2
- вариант ответа 3 и так далее
?
Ответ краткий и емкий
Дополнительное развернутое обоснование, если в нем есть острая необходимость
Источник: название заметки.md

Вопрос следующей flashkard...
```

## Структура проекта 

Файлы с тестами находятся в директории /Tests. Внутри могут быть созданы отдельные директории для групп файлов с falshcards на общую тему.

Называй создаваемые .md-файлы всегда в нижнем регистре, разделяя отдельные слова дефисами, в виде "тема-подтема.md".

## Правила генерации

- Язык повествования всегда русский, понятия и термины пиши на английском.
- Ты можешь не добавлять подтемы в тег flashcards если тема общая и обширная.
- Проверяй, чтобы при генерации flashcars вопрос действительно всегда содержал весь контекст, передающий суть вопроса.
- Если название нового файла не задано в запросе, то анализируй существующие варианты названий файлов в /Tests и на их основе генерируй подходящие варианты названий, предлагай выбрать один из них или вписать свой.
- Если тесты объединены общей темой, тогда создавай папку для новых flascars-файлов внутри /Tests - предлагай на выбор варианты названия новой папки, если я не задал конкретное в запросе.
- После генерации текста для файла пиши краткий отчет о создании flashcars: тема, папка, количество вопросов и запрашивай подтверждение на создание или обновление файла.

# Obsidian Flashcards Builder Skill (Spaced Repetition)

Создавай новые файлы с flashcards или дорабатывай существующие для работы через плагин **Obsidian Spaced Repetition**.

Триггером для использования скилла является упоминание flashcards, Spaced Repetition или просто тестов, которые необходимо создать или исправить.

## Структура создаваемых Flash Cards

Синтаксис создания заметок должен быть следующим: хештег #flashcards в начале файла, точный и конкретный вопрос, знак вопроса, ответ, пропуск строки, следующий вопрос.

Не допускай появления пустых строк внутри пары вопрос-ответ - пропуск строки только для разделения вопросов. Внутри вопросов используй только перенос на следующую строку. Пример синтаксиса:

```
## Тест по теме Example

#flashcards/тема/подтема

Текст вопроса, максимально однозначный понятный
- если необходимо, вариант ответа
- вариант ответа 2
- вариант ответа 3 и так далее
?
Ответ краткий и емкий
Дополнительное развернутое обоснование, если в нем есть острая необходимость
Источник: Obsidian Vault/Folder A/Folder B/Folder C/название заметки.md

Вопрос следующей flashkard...
```

## Структура проекта

Файлы с тестами находятся в директории /Tests. Называй создаваемые .md-файлы всегда в нижнем регистре, разделяя отдельные слова дефисами, в виде "тема-подтема.md". Тема и подтема в назввании файла не обязательно должны соответствовать теме и подтеме, указанным в хештеге в начале файла.

## Правила генерации

- Проверяй, чтобы при генерации flashcars вопрос действительно всегда содержал весь контекст, передающий суть вопроса.
- Если название нового файла не задано в запросе, то анализируй существующие варианты названий файлов в /Tests и на их основе генерируй подходящие варианты названий, если решения о выборе названия не выглядит очевидным, тогда предлагай выбрать один из них или вписать свой. Точно также поступай с выбором текста для хештега, но старайся анализировать и делать канонично в одном стиле с существующими файлами.
- В сообщении о выполнении задачи пиши краткий отчет о создании flashcars: тема, хештег, количество вопросов.
- При обновлении файлов с вопросами удаляй методанные формата <!--SR:!2026-04-07,3,250-->, оставляй между парами вопрос-отовет только пустую строку.

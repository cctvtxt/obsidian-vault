# Руководство по Стилю для Базы Знаний (GEMINI.md)

Этот документ описывает актуальные правила написания и структурирования заметок. Ключевой принцип — читабельность и плавность повествования.

---
## Общие Правила Форматирования

**Табуляция**: Никогда не используй символы табуляции и тройные пробелы. Форматирование отступами запрещено.

**Списки**: Не используй маркированные списки (`-`, `*`). Вместо них формируй полноценные, самодостаточные абзацы.

## Общие Правила Структуры

**Стиль изложения**: Пиши по существу, избегая вводных фраз и "воды". Текст должен быть плотным по смыслу, но изложен простым и понятным языком. Каждый абзац должен полностью раскрывать один конкретный аспект темы, обеспечивая исчерпывающее освещение вопроса без лишних слов.

**Заголовки**: Избегай избыточного количества заголовков. Вместо множества мелких секций, объединяй связанные мысли в развернутые абзацы, чтобы создать единый, плавный текст.

**Вступительное Описание**: Каждая заметка должна начинаться с абзаца, дающего определение или краткое описание темы. Не используй для этого специальный заголовок вроде `## Описание`.

**Ссылки**: Ссылка на родительскую концепцию (например, `[[passport]]`) размещается **только в самом начале файла**, перед любым другим текстом. Ссылки в основном тексте не допускаются.

---
## Правила для npm-библиотек (./NPM Registry)

**Полное Отсутствие Заголовков**: В файлах, описывающих NPM-библиотеки (например, стратегии Passport), заголовки (`##`) не используются вовсе. Весь текст должен представлять собой единое повествование, разделенное на абзацы и блоки кода.

**Структура Повествования**: Для NPM-библиотек текст должен последовательно раскрывать следующие аспекты, изложенные в свободной форме: основное назначение и принцип работы библиотеки, пример интеграции в виде блока кода, и развернутый анализ применимости, целесообразности, сравнение с альтернативами и важные аспекты использования.

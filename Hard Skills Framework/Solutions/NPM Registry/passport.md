[[Authentification]]

**Passport** — это гибкий, модульный промежуточный слой (middleware) для аутентификации в приложениях Node.js, построенный на базе Connect/Express. Его ключевая особенность — стратегический подход: ядро библиотеки не реализует конкретные механизмы входа, а предоставляет унифицированный интерфейс для подключения более чем 500 стратегий (`passport-local`, `passport-jwt`, `passport-google-oauth20` и др.).

Задача `passport` — стандартизировать процесс аутентификации: инициализацию, обработку запросов, сериализацию пользователя в сессию и восстановление данных при последующих запросах. Библиотека абстрагирует разработчика от низкоуровневых деталей, позволяя сосредоточиться на бизнес-логике, при этом сохраняя полный контроль над потоком аутентификации.

Пример ниже демонстрирует минимальную, но полноценную настройку ядра `passport` в приложении Express с использованием сессионной аутентификации:

|Компонент|Назначение|
|---|---|
|`passport.initialize()`|Подключает middleware для обработки аутентификации в каждом запросе|
|`passport.session()`|Интегрируется с `express-session` для восстановления пользователя из сессии|
|`serializeUser`|Определяет, какие данные пользователя сохранять в сессии (обычно `user.id`)|
|`deserializeUser`|Определяет, как загрузить полный объект пользователя по данным из сессии|
|`req.login()` / `req.logout()`|Программное управление состоянием аутентификации|
|`req.isAuthenticated()`|Хелпер для проверки, аутентифицирован ли текущий запрос|



Надежность - способность API выполнять заявленные функции без сбоев в течение заданного времени

| арактеристика                         | Описание                                          | Метрики                                            | Механизмы обеспечения                                 |
| ------------------------------------- | ------------------------------------------------- | -------------------------------------------------- | ----------------------------------------------------- |
| **Availability (Доступность)**        | % времени, когда API доступен для использования   | Uptime %, SLA (99.9%/99.99%)                       | Multi-AZ, active-active, health checks, auto-failover |
| **Correctness (Корректность)**        | Возврат точных и валидных результатов             | % успешных валидаций, ошибка бизнес-логики         | Schema validation, contract testing, idempotency      |
| **Data Consistency**                  | Согласованность данных между запросами            | Eventual consistency lag, transaction success rate | ACID-транзакции, sagas, outbox pattern                |
| **Mean Time Between Failures (MTBF)** | Среднее время безотказной работы                  | Часы/дни между инцидентами                         | Proactive monitoring, preventive maintenance          |
| **Mean Time To Recovery (MTTR)**      | Среднее время восстановления после сбоя           | Минуты до полного восстановления                   | Automated rollback, runbooks, chaos testing           |
| **Idempotency**                       | Повторный идентичный запрос даёт тот же результат | % idempotent endpoints, duplicate processing rate  | `Idempotency-Key`, deduplication, safe HTTP methods   |


**Надёжность** — это комплексное свойство системы сохранять во времени способность выполнять требуемые функции в заданных режимах и условиях применения. Она включает в себя такие аспекты, как безотказность, долговечность, ремонтопригодность, сохраняемость и другие. Надёжность отражает способность системы избегать системных отказов, которые являются по своей частоте и серьёзности неприемлемыми для пользователей. Количественно надёжность часто оценивают с помощью показателя среднего времени наработки на отказ (MTBF — Mean Time Between Failure).

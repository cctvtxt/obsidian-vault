[[API Compliance]]

PII (Personally Identifiable Information) — это не конкретный стандарт, а общая концепция, обозначающая любую информацию, которая может быть использована для идентификации конкретного человека. Понятие PII лежит в основе большинства регуляторных актов о защите данных, таких как GDPR, 152-ФЗ и CCPA.

Концепция PII решает задачу определения, какие именно данные подлежат защите. Она помогает компаниям понять, какая информация является чувствительной и требует применения специальных мер безопасности при ее обработке через API.

PII включает в себя как прямые идентификаторы (имя, номер паспорта, ИНН), так и косвенные (адрес электронной почты, номер телефона, IP-адрес, геолокация, данные cookie), которые в совокупности могут привести к установлению личности. Классификация данных как PII является первым шагом к построению системы их защиты.

# Вводимые ограничения

API должен быть спроектирован с полным пониманием того, какие из передаваемых данных являются PII. Для таких данных должен применяться принцип **минимизации** — запрашивать и обрабатывать только абсолютно необходимый объем информации.

Любые эндпоинты, передающие или обрабатывающие PII, должны быть защищены строгими мерами контроля доступа (аутентификация, авторизация) и использовать шифрование (TLS) для защиты данных при передаче.

Система должна иметь четкий **data governance** — политику управления данными, которая определяет, где и как PII хранятся, кто имеет к ним доступ, и как обеспечивается их жизненный цикл, включая безопасное удаление по истечении срока необходимости. В логах и системах мониторинга PII должны маскироваться или полностью исключаться.

# Практический путь внедрения

Правильная работа с PII — это фундамент для соответствия любым регуляторным требованиям. Первым шагом является проведение **инвентаризации данных** (data discovery and classification), чтобы точно определить, какие API и системы работают с PII.

На основе этой классификации выстраивается архитектура безопасности. Разработка API усложняется, так как для полей с PII необходимо внедрять шифрование на уровне приложения, токенизацию или маскирование. Это трудоемкий процесс, но он позволяет снизить риски утечек, избежать штрафов и построить систему, соответствующую требованиям `Privacy by Design`.

[[API Compliance]]

PCI DSS (Payment Card Industry Data Security Standard) — это стандарт безопасности данных индустрии платежных карт, разработанный международными платежными системами (Visa, MasterCard, American Express и др.). Он обязателен для всех компаний, которые хранят, обрабатывают или передают данные держателей карт.

Стандарт решает задачу защиты данных платежных карт от мошенничества и утечек путем установления единых требований к безопасности IT-инфраструктуры.

PCI DSS состоит из 12 детализированных требований, сгруппированных в шесть целей. Они охватывают все аспекты безопасности: от построения и сопровождения защищенной сети и защиты данных держателей карт до внедрения строгих мер контроля доступа, регулярного мониторинга и тестирования сетей, а также поддержания политики информационной безопасности.

# Вводимые ограничения

API, через который проходят данные платежных карт, должен находиться в изолированном и защищенном сегменте сети (CDE - Cardholder Data Environment). Доступ к этому сегменту должен быть строго ограничен как на сетевом, так и на прикладном уровне.

Запрещается хранить чувствительные аутентификационные данные (SAD), такие как полный трек магнитной полосы, CAV2/CVC2/CVV2/CID, PIN/PIN-блок, после авторизации. Номер карты (PAN) должен храниться в зашифрованном или токенизированном виде. API не должен возвращать в открытом виде полный номер карты, допустимо только маскирование (например, `4111...1111`).

Все компоненты системы, включая API, должны проходить регулярное сканирование на уязвимости (ASV-сканирование) и тесты на проникновение (пентесты). Код API должен разрабатываться в соответствии с практиками безопасной разработки и проходить статический (SAST) и динамический (DAST) анализ безопасности.

# Практический путь внедрения

Соответствие PCI DSS — это непрерывный процесс, а не разовый проект. Внедрение стандарта требует значительных инвестиций в безопасную архитектуру, сертифицированные решения (например, WAF, SIEM) и регулярный аудит.

Для API-ориентированных систем популярным решением является использование **токенизации** от сертифицированных провайдеров (PSP). Это позволяет вывести API из скоупа PCI DSS: вместо реальных данных карты API оперирует безопасными токенами, а вся работа с PAN ложится на плечи провайдера. Такой подход значительно снижает сложность, стоимость и риски, связанные с самостоятельной сертификацией.

[[API Compliance]]

CCPA (California Consumer Privacy Act) и его расширение CPRA (California Privacy Rights Act) — это калифорнийские законы о защите персональных данных, которые являются одними из самых строгих в США. Они применяются к компаниям, ведущим бизнес в Калифорнии и работающим с данными ее резидентов, устанавливая требования к обработке и защите этой информации через API.

Основная задача этих законов — предоставить потребителям контроль над их персональными данными. Это включает право знать, какие данные о них собираются, право на удаление данных и право запретить их продажу.

Стандарты определяют широкий спектр персональной информации, вводят понятия «продажи» и «обмена» данными, а также устанавливают конкретные обязанности для бизнеса. Они охватывают такие темы, как прозрачность (через политики конфиденциальности), механизмы реализации прав потребителей и требования к кибербезопасности.

# Вводимые ограничения

API должен предоставлять конечным пользователям четкие и доступные механизмы для реализации их прав. Это включает эндпоинты для запроса на доступ к своим данным (`data access request`), их удаление (`deletion request`) и отказ от их продажи (`opt-out`).

Система должна быть спроектирована так, чтобы минимизировать сбор данных (**data minimization**) и ограничить их использование только заявленными целями (**purpose limitation**). API не должен собирать или передавать данные, которые не являются абсолютно необходимыми для предоставления услуги.

Требуется внедрение «разумных мер безопасности» для защиты данных. Для API это означает использование надежного шифрования (в покое и при передаче), строгую аутентификацию и авторизацию, а также регулярный аудит безопасности и оценку рисков.

# Практический путь внедрения

Внедрение CCPA/CPRA требует значительных изменений в архитектуре API и бизнес-процессах. Необходимо не только создать публичные эндпоинты для запросов от пользователей, но и обеспечить сквозную обработку этих запросов во всех внутренних системах, где хранятся данные.

Разработка усложняется необходимостью точно классифицировать данные, чтобы отличать персональную информацию от неперсональной, и отслеживать потоки данных для выполнения запросов на удаление или отказ от продажи. Хотя это увеличивает сложность и стоимость, соблюдение этих стандартов является обязательным для работы с калифорнийскими потребителями и помогает укрепить доверие пользователей.

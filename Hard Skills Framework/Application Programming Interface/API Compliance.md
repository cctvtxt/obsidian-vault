[[API]]

Compliance является характеристикой API, описывающей способность API соответствовать регуляторным, отраслевым и внутренним стандартам. Требования к API‑compliance формируются на основе законодательства, регуляторных актов, отраслевых стандартов и договорных обязательств.

1. **Регуляторное соответствие**  
Соблюдение законов и регуляторных стандартов, включая GDPR, HIPAA, PCI‑DSS, SOX, 152‑ФЗ РФ, CCPA, FISMA, PSD2; аудиты пройдены, нет штрафов, документированные DPA/BAA/Соглашения.
2. **Стандарты данных**  
Использование JSON Schema, OpenAPI 3.1, ISO 8601, UTF‑8, XML Schemas и Protobuf contracts; валидация по схеме, консистентность форматов, типизация всех полей.
3. **Аудит и трассируемость**  
Фиксация всех критических операций через API; 100% критических операций залогировали, retention ≥ N дней, интеграция с SIEM; audit logs, change‑logs, immutable storage, correlation ID.
4. **Управление согласием**  
Контроль обработки персональных данных через API; явное согласие, право на удаление, отказ от обработки; consent flags, DELETE `/users/{id}`, анонимизация данных, согласия в IAM‑системах.
5. **Гео‑ограничения**  
Соблюдение требований к локализации и маршрутизации данных; данные не покидают регион, маршрутизация по юрисдикции, региональные ЦОД; data residency policies, region‑aware routing, geo‑zone‑aware API‑шлюзы.
6. **Лицензирование и квоты**  
Контроль использования API по тарифным и регуляторным ограничениям; продукт/клиенты привязаны к тарифам, блокировка при превышении; usage meters, rate‑limiting, quota headers, billing API, usage‑condition‑based API.

В России API‑compliance — это не только техническая, но и юридическая обязанность. Банк России, ФСБ, ФСТЭК и Роскомнадзор требуют от компаний обеспечить защиту данных, аудит, локализацию и соответствие регуляторным стандартам. В российских финансовых сервисах это означает внедрение Open API‑стандартов, платформы управления согласиями (ПКС) и интеграцию с государственными системами. В международных проектах — это соответствие GDPR, HIPAA, PCI‑DSS и CCPA/CPRA.

[[Key-Based Auth]]

**Ключ API (API Key)** — это уникальная строка символов, используемая для идентификации и аутентификации приложения (или пользователя) при выполнении запросов к API. Это простой механизм контроля доступа, который не требует сложного обмена токенами, как в OAuth.
___
## Генерация

Ключи всегда генерируются на стороне сервера. Обычно это происходит, когда разработчик регистрирует свое приложение в сервисе. Ключ должен быть длинной, криптографически случайной строкой.
```javascript
const apiKey = crypto.randomBytes(32).toString('hex');
```
Клиент (приложение-потребитель API) не генерирует, а только получает и использует ключ, выданный сервером.
## Хранение

При получении запроса сервер хеширует входящий ключ и сравнивает его с хешем в базе данных. Это защищает от захвата всех ключей при утечке базы.
```sql
SELECT user_id FROM api_keys WHERE key_hash = SHA256('provided_api_key');
```
**В системе клиента** ключ API должен рассматриваться как пароль. Его следует хранить в `переменных окружения`, `.env` файле или через систему управления секретами.
## Передача

Передача ключа **обязательно** должна происходить по зашифрованному каналу (`HTTPS`).

Существует несколько распространенных способов:
+ **Пользовательский HTTP-заголовок (рекомендуется)**:
```http
GET /api/data
Host: example.com
X-API-Key: <your_api_key>
```
+ **Заголовок `Authorization`**:
```http
GET /api/data
Host: example.com
Authorization: ApiKey <your_api_key>
```
+ **Параметр запроса (менее безопасно)**:
```http
GET /api/data?apiKey=<your_api_key>
```
Этот способ не рекомендуется, так как URL-адреса, включая параметры, часто логируются и могут попасть в историю браузера.

[[Auth]]

Заголовок запроса `Proxy-Authorization` используется для аутентификации клиента на промежуточном прокси-сервере. Он аналогичен заголовку `Authorization`, но предназначен специально для прокси.

# Описание

Если прокси-сервер требует аутентификации для пропуска запроса к целевому серверу, он отвечает клиенту статусом `407 Proxy Authentication Required`. В ответ клиент должен повторить запрос, добавив в него заголовок `Proxy-Authorization` с учетными данными.

Синтаксис идентичен `Authorization`: `Proxy-Authorization: <type> <credentials>`

Схемы аутентификации (`Basic`, `Digest` и т.д.) также аналогичны тем, что используются в `Authorization`.

# Юзкейсы

*   **Корпоративные сети**: В компаниях часто используются прокси-серверы для контроля доступа в интернет. Сотрудникам необходимо аутентифицироваться на прокси, чтобы выходить в сеть.
*   **Платные прокси-сервисы**: Сервисы, предоставляющие доступ к прокси-пулам, используют этот заголовок для идентификации платных пользователей.
*   **Контроль доступа**: Ограничение доступа к определенным ресурсам через прокси на основе учетных данных пользователя.

# Проблемы и особенности

*   **Сквозная безопасность**: Учетные данные для `Proxy-Authorization` предназначены только для прокси и не должны передаваться конечному серверу. Прокси-сервер должен удалить этот заголовок перед отправкой запроса дальше.
*   **Конфигурация клиента**: Не все HTTP-клиенты и библиотеки по умолчанию поддерживают или корректно настраивают прокси-аутентификацию, что может потребовать дополнительной конфигурации.
*   **Множественные прокси**: В цепочке из нескольких прокси каждый может требовать свою аутентификацию, что усложняет процесс. Заголовок `Proxy-Authorization` предназначен для следующего (ближайшего) прокси в цепочке.

[[HTTP Headers]]

Заголовки **аутентификации и авторизации** используются для проверки подлинности клиента, запрашивающего доступ к защищенному ресурсу. Они позволяют серверу определить, кто делает запрос и какими правами он обладает.

1. `Authorization` — передаёт учётные данные или токен, например `Bearer`.
2. `WWW-Authenticate` — сервер сообщает, какой способ аутентификации нужен.
3. `Proxy-Authorization` — авторизация клиента перед прокси.
4. `Proxy-Authenticate` — прокси сообщает, как нужно авторизоваться.
5. `X-API-Key` — кастомный ключ доступа к API.
6. `X-Auth-Token` — кастомный токен для доступа к ресурсу.
7. `Auth-Token` — альтернативный кастомный заголовок для токена.
8. `X-Access-Token` — кастомный access token для API.
9. `X-Login-Token` — кастомный заголовок для входа или одноразовой проверки.
10. `X-Credential` — общий кастомный заголовок для передачи credentials.

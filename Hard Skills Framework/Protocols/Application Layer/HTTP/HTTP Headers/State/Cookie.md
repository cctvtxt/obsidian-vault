`Cookie` — это заголовок HTTP-запроса, который содержит `cookie`, ранее установленные сервером для данного домена. Это основной механизм, позволяющий серверам "запоминать" информацию о клиенте между запросами.

# Описание

Когда сервер хочет сохранить какую-то информацию на стороне клиента, он отправляет заголовок `Set-Cookie`. Браузер сохраняет эту информацию и при последующих запросах к этому же серверу автоматически прикрепляет ее в заголовке `Cookie`.

Значение заголовка — это список пар `имя=значение`, разделенных точкой с запятой и пробелом.

`Cookie: PHPSESSID=298zf09hf012fh2; theme=dark; logged_in=true`

# Юзкейсы

*   **Управление сессиями**: Самый частый юзкейс. После успешного входа сервер устанавливает `cookie` с уникальным идентификатором сессии. При каждом последующем запросе клиент отправляет этот ID, позволяя серверу идентифицировать пользователя и поддерживать его аутентифицированное состояние.
*   **Персонализация**: Сохранение пользовательских настроек, таких как тема сайта (темная/светлая), выбранный язык или местоположение.
*   **Отслеживание и аналитика**: Сервисы аналитики (Google Analytics, Яндекс.Метрика) используют `cookie` для отслеживания уникальных посетителей, подсчета визитов и анализа поведения пользователей на сайте.

# Проблемы и особенности

*   **Безопасность**:
    *   `cookie` передаются с каждым HTTP-запросом к домену, что может создавать избыточный трафик.
    *   `cookie` могут быть перехвачены при использовании `HTTP` без шифрования. Атрибут `Secure` в `Set-Cookie` решает эту проблему.
    *   Межсайтовая подделка запроса (CSRF) — это атака, эксплуатирующая автоматическую отправку `cookie` браузером. Для защиты используется CSRF-токен или атрибут `SameSite`.
*   **Хранение на клиенте**: `cookie` хранятся на машине пользователя в виде простого текста, поэтому в них никогда нельзя хранить чувствительную информацию (пароли, номера карт).
*   **Доступ из JavaScript**: По умолчанию `cookie` доступны из JavaScript (`document.cookie`), что делает их уязвимыми для XSS-атак. Атрибут `HttpOnly` в `Set-Cookie` запрещает доступ к `cookie` из JS.
*   **Размер и количество**: Браузеры накладывают ограничения на размер (обычно до 4 КБ) и количество `cookie` на один домен (обычно 20-50).

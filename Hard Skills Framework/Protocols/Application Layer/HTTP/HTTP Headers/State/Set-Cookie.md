`Set-Cookie` — это заголовок HTTP-ответа, который сервер использует для отправки `cookie` на клиент (в браузер) для последующего хранения.

# Описание

Значение заголовка состоит из пары `имя=значение` и нескольких опциональных атрибутов, которые контролируют поведение `cookie`.

`Set-Cookie: <cookie-name>=<cookie-value>; Expires=<date>; Path=<path>; Domain=<domain>; Secure; HttpOnly; SameSite=<Strict|Lax|None>`

**Основные атрибуты:**

1.  `Expires=<date>`: Устанавливает дату истечения срока действия `cookie`. После этой даты `cookie` удаляется. Если не указан (и нет `Max-Age`), `cookie` становится сессионным и удаляется при закрытии браузера.
2.  `Max-Age=<seconds>`: Устанавливает срок жизни `cookie` в секундах. Имеет приоритет над `Expires`.
3.  `Domain=<domain>`: Указывает, для какого домена действителен `cookie`. Если не указан, используется домен текущего сервера. Можно указать родительский домен (например, `.example.com`), чтобы `cookie` был доступен на всех его поддоменах.
4.  `Path=<path>`: Указывает путь внутри домена, для которого действителен `cookie` (например, `/api`). `cookie` будет отправляться только при запросах к этому пути и его подпутям.
5.  `Secure`: Флаг. Если он установлен, `cookie` будет отправляться только по `HTTPS`-соединению.
6.  `HttpOnly`: Флаг. Если он установлен, `cookie` будет недоступен для чтения из JavaScript (`document.cookie`), что является ключевой мерой защиты от XSS-атак.
7.  `SameSite=<Strict|Lax|None>`: Контролирует, когда `cookie` должен отправляться при кросс-сайтовых запросах. Основной механизм защиты от CSRF-атак.
    *   `Strict`: `cookie` отправляется только при запросах с того же сайта.
    *   `Lax`: `cookie` отправляется при навигации на ваш сайт с другого сайта (например, по ссылке), но не при фоновых запросах (`fetch`, `<img>`). Значение по умолчанию в современных браузерах.
    *   `None`: `cookie` отправляется при любых кросс-сайтовых запросах. Требует обязательного использования атрибута `Secure`.

# Юзкейсы

*   **Создание сессии**: Установка сессионного `cookie` после входа пользователя.
*   **Долгоживущая авторизация**: Установка `cookie` с `Expires` или `Max-Age` для функции "Запомнить меня".
*   **Установка настроек**: Сохранение предпочтений пользователя.

# Проблемы и особенности

*   **Множественные `Set-Cookie`**: Сервер может отправить несколько заголовков `Set-Cookie` в одном ответе для установки нескольких `cookie`.
*   **Конфигурация безопасности**: Правильная комбинация атрибутов `Secure`, `HttpOnly` и `SameSite` критически важна для безопасности современного веб-приложения.

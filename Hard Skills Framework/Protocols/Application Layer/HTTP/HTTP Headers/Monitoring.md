[[HTTP Headers]]

Заголовки для **мониторинга и отладки** предоставляют контекст о запросе, который полезен для сбора метрик, логирования и распределенной трассировки. Они помогают отслеживать путь запроса через несколько сервисов и анализировать его поведение.

1. `User-Agent` — описывает клиентское приложение или браузер.
2. `Referer` — показывает страницу-источник запроса.
3. `X-Request-ID` — уникальный ID запроса для логов.
4. `X-Correlation-ID` — связывает события одного сценария.
5. `Traceparent` — стандартный заголовок распределённой трассировки.
6. `Tracestate` — дополнительное состояние трассировки.
7. `X-B3-TraceId` — trace ID в формате Zipkin/B3.
8. `X-B3-SpanId` — span ID в формате Zipkin/B3.
9. `X-B3-Sampled` — флаг, нужно ли сэмплировать трассу.
10. `X-Custom-Log-Tag` — кастомная метка для логирования.

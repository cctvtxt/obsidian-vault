[[Monitoring]]

`traceparent` — это стандартный заголовок запроса из спецификации W3C Trace Context, предназначенный для сквозной распределенной трассировки запросов в микросервисной архитектуре.

# Описание

Заголовок имеет строго определенный формат и содержит четыре части, разделенные дефисами:

`traceparent: <version>-<trace-id>-<parent-id>-<trace-flags>`

*   `version` (2 символа): Версия стандарта (например, `00`).
*   `trace-id` (32 символа): Уникальный идентификатор всей цепочки вызовов (всего сквозного запроса). Генерируется на самом первом сервисе.
*   `parent-id` (16 символов): Идентификатор родительского спана (span). Для каждого отдельного вызова внутри цепочки генерируется новый `id`, и он становится `parent-id` для следующего вызова.
*   `trace-flags` (2 символа): Флаги, например, флаг `sampled`, который указывает, нужно ли записывать данные этой трассировки в систему мониторинга.

# Юзкейсы

*   **Распределенная трассировка**: Это основной и единственный юзкейс. `traceparent` позволяет системам мониторинга (таким как Jaeger, Zipkin, OpenTelemetry) связать воедино логи, метрики и спаны от разных сервисов, относящиеся к одному и тому же пользовательскому запросу.
*   **Визуализация пути запроса**: На основе данных трассировки можно строить "водопадные" диаграммы (waterfall charts), которые показывают, сколько времени заняла обработка в каждом сервисе, и быстро находить узкие места.
*   **Анализ ошибок**: Позволяет увидеть всю картину целиком при анализе ошибки, которая произошла в глубине системы.

# Проблемы и особенности

*   **Стандартизация**: `traceparent` — это современный стандарт, который приходит на смену проприетарным и нестандартным заголовкам (`X-Request-ID`, `X-B3-TraceId` и т.д.). Рекомендуется использовать именно его.
*   **Экосистема**: Требует поддержки со стороны всех компонентов системы: клиентов, серверов, балансировщиков, API Gateway. Современные фреймворки и библиотеки для мониторинга поддерживают его "из коробки".
*   **`tracestate`**: В паре с `traceparent` может идти заголовок `tracestate`, который несет дополнительную, специфичную для вендора информацию о трассировке. `traceparent` содержит универсальные данные, а `tracestate` — кастомные.
*   **Сэмплирование**: Флаг `sampled` позволяет управлять объемом собираемых данных. Можно, например, записывать только 1% всех трассировок, чтобы не перегружать систему мониторинга.

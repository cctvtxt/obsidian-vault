[[Monitoring]]

Заголовок запроса `User-Agent` содержит строку, которая идентифицирует клиентское приложение, его операционную систему, вендора и/или версию.

# Описание

Это одна из самых известных и хаотичных строк в вебе. Исторически сложилось, что для совместимости она содержит в себе информацию о других браузерах.

Пример `User-Agent` из Chrome в Windows:
`User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/108.0.0.0 Safari/537.36`

Здесь можно увидеть `Mozilla/5.0`, `AppleWebKit`, `KHTML, like Gecko`, `Chrome` и `Safari` — всё в одной строке.

# Юзкейсы

*   **Веб-аналитика**: Сбор статистики о том, какие браузеры, операционные системы и устройства используют посетители сайта. Это помогает принимать решения о поддержке тех или иных технологий.
*   **Адаптивный контент**: Сервер может анализировать `User-Agent` для определения типа устройства (мобильное, десктоп) и отдавать соответствующую версию сайта. Однако для этого сейчас предпочтительнее использовать адаптивный дизайн (media queries) на клиенте.
*   **Блокировка или специальная обработка ботов**: Поисковые роботы (например, `Googlebot`) и другие краулеры обычно имеют уникальный `User-Agent`. Это позволяет серверу отдавать им специальный контент или, наоборот, блокировать нежелательных ботов.
*   **Исправление багов**: Иногда для обхода специфичных багов в старых браузерах сервер может отдавать им измененный CSS или JS.

# Проблемы и особенности

*   **Ненадежность и подделка**: `User-Agent` легко подделать. Любой HTTP-клиент может установить произвольное значение этого заголовка.
*   **Сложность парсинга**: Из-за хаотичной структуры парсинг `User-Agent` — нетривиальная задача. Для этого всегда следует использовать специализированные библиотеки.
*   **Приватность и `User-Agent Client Hints`**: Строка `User-Agent` может использоваться для снятия "цифрового отпечатка" пользователя (fingerprinting). Для борьбы с этим и для упрощения получения информации браузеры (во главе с Chrome) внедряют новый механизм — User-Agent Client Hints. Он позволяет серверу запрашивать только нужную информацию о клиенте (например, бренд браузера, версию, тип устройства) через отдельные заголовки (`Sec-CH-UA`, `Sec-CH-UA-Mobile` и т.д.), вместо того чтобы получать одну большую и избыточную строку.

[[Monitoring]]

`tracestate` — это заголовок запроса из спецификации W3C Trace Context, который используется в паре с заголовком `traceparent`. Он несет дополнительную, специфичную для вендора или системы информацию в рамках распределенной трассировки.

# Описание

Если `traceparent` содержит универсальный и стандартизированный контекст трассировки (ID трассы, ID спана), то `tracestate` представляет собой список пар `ключ=значение`, где разные системы мониторинга могут хранить свои собственные данные.

`tracestate: congo=t61rcWkgMzE, rovio=00f067aa0ba902b7`

В этом примере:
*   Система `congo` сохранила свои данные (`t61rcWkgMzE`).
*   Система `rovio` сохранила свои (`00f067aa0ba902b7`).

Каждый сервис в цепочке может добавлять или обновлять свою часть `tracestate`, но не должен трогать данные других систем.

# Юзкейсы

*   **Межвендорная совместимость**: Позволяет разным системам мониторинга (например, от разных поставщиков облачных услуг) обмениваться информацией в рамках одной трассы. Система `A` может передать какие-то свои метаданные системе `B`.
*   **Передача дополнительного контекста**: Можно передавать информацию, которой нет места в `traceparent`, например, ID клиента, приоритет запроса или информацию для сэмплирования.
*   **Координация сэмплирования**: Разные системы в цепочке могут использовать `tracestate` для согласования решения о том, следует ли сэмплировать (записывать) данную трассировку.

# Проблемы и особенности

*   **Сложность**: `tracestate` сложнее, чем `traceparent`. Его формат и правила обработки требуют внимательной реализации.
*   **Не является обязательным**: В отличие от `traceparent`, заголовок `tracestate` является опциональным.
*   **Ограничение размера**: Спецификация накладывает ограничение на общий размер заголовка (обычно 512 символов) и количество пар ключ-значение (до 32), чтобы не раздувать заголовки запросов.
*   **Непрозрачность**: Данные в `tracestate` обычно являются "непрозрачными" для сервисов, которые не являются их владельцами. Сервис должен только передавать их дальше, не пытаясь интерпретировать.

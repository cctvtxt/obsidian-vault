[[HTTP Headers]]

Заголовки **управления состоянием** используются для отслеживания сеанса взаимодействия между клиентом и сервером. Поскольку HTTP сам по себе является протоколом без сохранения состояния, эти заголовки позволяют "запоминать" информацию о пользователе между запросами.

1. `Cookie` — клиент отправляет сохранённые cookie на сервер.
2. `Set-Cookie` — сервер устанавливает cookie в браузере.
3. `X-Session-Token` — кастомный сессионный идентификатор.
4. `Session-Id` — кастомный идентификатор сессии.
5. `X-Client-Session` — кастомный токен клиентской сессии.
6. `X-State-Token` — кастомный маркер состояния пользователя.
7. `Remember-Me` — часто используется для долгоживущей авторизации/сессии.
8. `X-Browser-State` — кастомный маркер состояния браузера.
9. `X-User-Session` — кастомный заголовок сессии пользователя.
10. `State-Token` — общий кастомный токен состояния.

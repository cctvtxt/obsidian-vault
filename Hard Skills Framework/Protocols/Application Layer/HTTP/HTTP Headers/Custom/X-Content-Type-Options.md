[[Custom]]

`X-Content-Type-Options` — это заголовок ответа, который используется для защиты от атак, связанных с "угадыванием" MIME-типа (MIME sniffing).

# Описание

Некоторые браузеры пытаются быть "умными" и игнорируют заголовок `Content-Type`, если считают, что реальный тип контента не соответствует заявленному. Например, если сервер отдал файл с `Content-Type: text/plain`, но его содержимое выглядит как `HTML` или `JavaScript`, браузер может попытаться исполнить его. Это может привести к XSS-атакам, если злоумышленнику удалось загрузить на сайт файл с вредоносным кодом под видом обычного текста.

Заголовок `X-Content-Type-Options` имеет единственное возможное значение:

`X-Content-Type-Options: nosniff`

Этот заголовок строго запрещает браузеру изменять `Content-Type`, указанный сервером.

# Юзкейсы

*   **Предотвращение XSS-атак**: Это основное предназначение заголовка. Он заставляет браузер доверять `Content-Type`, который прислал сервер. Если сервер говорит, что это `image/png`, браузер не будет пытаться исполнить этот файл как `JavaScript`, даже если его содержимое похоже на скрипт.
*   **Повышение безопасности**: Установка этого заголовка является одной из базовых рекомендаций по усилению безопасности веб-приложений. Он особенно важен для сайтов, которые позволяют пользователям загружать контент.

# Проблемы и особенности

*   **Широкая поддержка**: Заголовок поддерживается всеми современными браузерами.
*   **Простота**: В отличие от `Content-Security-Policy`, этот заголовок очень прост в реализации — достаточно добавить его во все ответы сервера с единственным значением `nosniff`.
*   **Требует корректного `Content-Type`**: Использование `X-Content-Type-Options: nosniff` подразумевает, что ваш сервер всегда отдает корректный заголовок `Content-Type`. В противном случае легитимный контент может перестать отображаться. Например, если вы отдаете CSS-файл с типом `text/plain`, браузер с `nosniff` не применит эти стили.

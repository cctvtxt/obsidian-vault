[[Custom]]

`X-Request-ID` (или `X-Correlation-ID`) — это нестандартный, но очень распространенный заголовок, который содержит уникальный идентификатор для конкретного запроса.

# Описание

Значением является уникальная строка (часто UUID или хэш), которая генерируется на клиенте или на первой точке входа в систему (например, на балансировщике нагрузки).

`X-Request-ID: f058ebd6-02f7-4d3f-942e-904344e8cde5`

Этот идентификатор затем передается (пробрасывается) через все микросервисы, которые участвуют в обработке данного запроса.

# Юзкейсы

*   **Распределенная трассировка и логирование**: Это основной юзкейс. Когда запрос проходит через несколько сервисов, наличие общего `X-Request-ID` во всех логах, относящихся к этому запросу, позволяет легко отследить весь его путь. Можно быстро найти все логи, связанные с одной операцией, даже если они разбросаны по десяткам систем.
*   **Отладка**: При возникновении ошибки можно попросить пользователя сообщить `Request ID` (если он отображается на странице ошибки), чтобы быстро найти в логах всю цепочку событий, которая к ней привела.
*   **Мониторинг**: Системы мониторинга могут использовать этот ID для группировки метрик и таймингов, относящихся к одному запросу.

# Проблемы и особенности

*   **Нестандартный**: Имя заголовка не стандартизировано. Часто используются альтернативы: `X-Correlation-ID`, `Request-ID`, `X-Amz-Request-Id` (в AWS). Важно придерживаться единого названия в рамках всей системы.
*   **Генерация ID**: ID должен быть достаточно уникальным. Обычно для этого используют UUID. Первая точка входа в систему (API Gateway, балансировщик) отвечает за его генерацию, если он не был предоставлен клиентом.
*   **Пробрасывание ID**: Каждый сервис в цепочке обязан читать `X-Request-ID` из входящего запроса и передавать его дальше во все исходящие запросы (к другим сервисам, базам данных и т.д.). Потеря этого заголовка на одном из этапов разрывает цепочку трассировки.
*   **Стандарт `Traceparent`**: В современных системах для распределенной трассировки рекомендуется использовать стандартный заголовок `Traceparent` из спецификации W3C Trace Context, который решает те же задачи, но является стандартом. `X-Request-ID` можно считать его предшественником.

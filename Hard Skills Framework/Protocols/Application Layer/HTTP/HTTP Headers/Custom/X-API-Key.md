[[Custom]]

`X-API-Key` — это нестандартный заголовок запроса, используемый для передачи простого ключа API для авторизации клиента.

# Описание

Ключ API (API key) — это секретный токен, который выдается пользователю или приложению для доступа к API. `X-API-Key` является одним из общепринятых, хотя и нестандартных, способов передачи этого ключа.

`X-API-Key: a1b2c3d4-e5f6-7890-1234-567890abcdef`

Сервер, получив запрос, извлекает ключ из заголовка, проверяет его валидность в своей базе данных, определяет, кому он принадлежит, и на основе этого решает, можно ли предоставить доступ.

# Юзкейсы

*   **Авторизация проектов/приложений**: Вместо аутентификации конкретного пользователя, ключ API часто идентифицирует проект или приложение, которое делает запрос. Это удобно для публичных API.
*   **Отслеживание использования**: Позволяет владельцам API отслеживать, кто и как часто использует их сервис. На основе этого можно выставлять счета (биллинг) или применять ограничения по количеству запросов (rate limiting).
*   **Простая защита**: Это простой способ защитить API от анонимного доступа, который легче реализовать, чем полноценный OAuth 2.0.

# Проблемы и особенности

*   **Нестандартный**: Имя заголовка не является стандартом. Встречаются альтернативы, такие как `Authorization: ApiKey <key>` или просто `Api-Key`.
*   **Безопасность**:
    *   Ключ передается в открытом виде, поэтому использование `HTTPS` является обязательным.
    *   Если ключ будет скомпрометирован, злоумышленник сможет делать запросы от имени его владельца.
    *   Ключи не должны храниться в открытом виде в клиентском коде (например, в JavaScript-файлах), так как их легко извлечь. Этот метод больше подходит для межсерверного взаимодействия.
*   **Управление ключами**: Требуется инфраструктура для генерации, отзыва и ротации ключей API.
*   **Не для гранулярных прав**: Ключ API обычно предоставляет доступ ко всему API или к его большой части. Для более гибкого управления правами (например, "разрешить этому пользователю читать, но не удалять") лучше подходит `OAuth 2.0`.

`X-XSS-Protection` — это заголовок ответа, который использовался для управления встроенным в старые браузеры механизмом защиты от отраженных XSS-атак (Reflected Cross-Site Scripting).

**Важно: Этот заголовок устарел и его использование больше не рекомендуется.** Современные браузеры удалили поддержку этой функции.

# Описание

Браузеры (в основном старые версии Chrome, Safari и Internet Explorer) имели встроенный фильтр, который пытался обнаружить, когда вредоносный скрипт, переданный в URL, "отражается" в HTML-ответе сервера. Заголовок `X-XSS-Protection` позволял управлять этим фильтром.

**Возможные значения:**

1.  `0`: Отключает XSS-фильтр.
2.  `1`: Включает XSS-фильтр. Если атака обнаружена, браузер "очистит" страницу, удалив вредоносный скрипт.
3.  `1; mode=block`: Включает фильтр. Если атака обнаружена, браузер полностью заблокирует отображение страницы вместо того, чтобы пытаться ее "очистить".
4.  `1; report=<reporting-URI>`: (Chromium) Включает фильтр и отправляет отчет о попытке атаки на указанный URI.

# Юзкейсы (Исторические)

*   **Дополнительный слой защиты**: В прошлом этот заголовок служил дополнительным, хотя и ненадежным, рубежом обороны от отраженных XSS-атак для пользователей старых браузеров.

# Проблемы и особенности

*   **УСТАРЕЛ**: Современные стандарты безопасности настоятельно рекомендуют отключать этот заголовок (`X-XSS-Protection: 0`) и полагаться на более надежный механизм — `Content-Security-Policy`.
*   **Создавал уязвимости**: Встроенные XSS-аудиторы сами могли создавать уязвимости. Например, они могли "ошибиться" и вырезать из страницы легитимный кусок кода, который случайно совпал с чем-то из URL, тем самым нарушая работу сайта или даже позволяя провести атаку.
*   **Неэффективность**: Фильтр защищал только от отраженных XSS и не спасал от хранимых (stored) или DOM-based XSS.
*   **Рекомендация OWASP**: Современные рекомендации OWASP (Open Web Application Security Project) — устанавливать `X-XSS-Protection: 0`, чтобы отключить этот небезопасный механизм, и использовать `Content-Security-Policy` в качестве основной защиты от XSS.

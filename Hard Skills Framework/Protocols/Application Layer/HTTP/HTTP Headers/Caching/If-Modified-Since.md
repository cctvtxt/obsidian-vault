[[Caching]]

Заголовок запроса `If-Modified-Since` делает запрос условным. Сервер вернет запрошенный ресурс, только если он был изменен после указанной даты. Если изменений не было, сервер вернет ответ `304 Not Modified`.

# Описание

Значением заголовка является дата в формате HTTP-date. Обычно браузер берет эту дату из заголовка ответа `Last-Modified`, который был получен вместе с ресурсом при предыдущем запросе.

**Процесс валидации:**

1.  Клиент запрашивает ресурс и получает его вместе с заголовком `Last-Modified: <дата>`.
2.  При следующем запросе того же ресурса клиент отправляет заголовок `If-Modified-Since: <дата>`.
3.  Сервер сравнивает дату из запроса с датой последнего изменения ресурса.
    *   Если ресурс не менялся, сервер отвечает `304 Not Modified`.
    *   Если ресурс был изменен, сервер отвечает `200 OK` с новой версией ресурса и новым `Last-Modified`.

# Юзкейсы

*   **Валидация кэша**: Наряду с `ETag`/`If-None-Match`, это основной механизм валидации кэша. Он позволяет клиентам проверять актуальность ресурса, не загружая его тело.
*   **Краулеры и боты**: Поисковые роботы могут использовать `If-Modified-Since` для экономии ресурсов, скачивая только обновленные страницы сайта.
*   **Синхронизация данных**: API-клиенты могут использовать этот заголовок для получения только тех данных, которые изменились с момента последнего запроса.

# Проблемы и особенности

*   **Точность времени**: Точность `Last-Modified` ограничена одной секундой. Если ресурс меняется несколько раз в секунду, этот механизм может дать сбой. `ETag` в этом плане надежнее.
*   **Не только для `GET`**: Заголовок может использоваться и с методом `HEAD`.
*   **Распределенные системы**: Как и в случае с `ETag`, в кластерной среде важно, чтобы дата `Last-Modified` была консистентной на всех серверах. Проблемы с синхронизацией времени могут нарушить кэширование.
*   **Игнорируется при наличии `If-None-Match`**: Если в запросе есть оба заголовка, `If-None-Match` имеет приоритет, а `If-Modified-Since` игнорируется. Это связано с тем, что `ETag` считается более точным механизмом валидации.

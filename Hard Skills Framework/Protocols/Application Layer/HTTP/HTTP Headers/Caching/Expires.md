[[Caching]]

Заголовок ответа `Expires` указывает дату/время, после которого ответ считается устаревшим. Это простейший механизм контроля кэширования.

# Описание

В качестве значения заголовок `Expires` содержит дату в формате HTTP-date. Например: `Expires: Wed, 21 Oct 2026 07:28:00 GMT`.

До наступления этой даты кэш (браузер или прокси) может отдавать ресурс без обращения к исходному серверу. После — ресурс считается устаревшим и требует валидации.

# Юзкейсы

*   **Кэширование статичных ресурсов**: Раньше `Expires` был основным способом кэшировать изображения, CSS и JS файлы на длительный срок.
*   **Простые сценарии**: Используется там, где не нужна сложная логика `Cache-Control`, а достаточно указать конкретный срок годности.

# Проблемы и особенности

*   **Приоритет `Cache-Control`**: Директива `max-age` в заголовке `Cache-Control` имеет более высокий приоритет, чем `Expires`. Если оба заголовка присутствуют в ответе, `Expires` будет проигнорирован.
*   **Синхронизация времени**: `Expires` использует абсолютную временную метку. Если часы на клиентской машине или прокси-сервере не синхронизированы с серверными, кэширование будет работать некорректно. `Cache-Control: max-age` решает эту проблему, так как использует относительное время.
*   **Устаревание**: Считается устаревшим механизмом в пользу `Cache-Control`, который предлагает гораздо большую гибкость. Тем не менее, его все еще полезно указывать для поддержки старых клиентов.
*   **Невалидная дата**: Дата в значении заголовка, равная 0 или -1, или невалидная дата, трактуется как "уже устарело".

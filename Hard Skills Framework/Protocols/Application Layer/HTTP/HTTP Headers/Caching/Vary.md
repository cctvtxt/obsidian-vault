[[Caching]]

Заголовок ответа `Vary` указывает, какие заголовки запроса сервер использовал для формирования ответа, помимо URL. Это инструкция для кэширующих серверов (браузера, CDN), что для разных значений этих заголовков нужно хранить разные версии ответа.

# Описание

Значением `Vary` является список имен заголовков запроса, разделенных запятыми.

`Vary: Accept-Encoding, User-Agent`

Это говорит кэшу: "Этот ответ зависит от заголовков `Accept-Encoding` и `User-Agent`. Если придет новый запрос с другим значением одного из этих заголовков, не отдавай ему эту кэшированную копию, а иди за новой на сервер".

# Юзкейсы

*   **Сжатие контента**: `Vary: Accept-Encoding`. Сервер может отдавать контент сжатым (`gzip`, `br`) или несжатым в зависимости от того, что поддерживает клиент (заголовок `Accept-Encoding`). `Vary` гарантирует, что клиент, не поддерживающий сжатие, не получит сжатую версию из кэша. Это самый частый юзкейс.
*   **Адаптивный контент**: `Vary: User-Agent`. Если сервер отдает разный HTML для мобильной и десктопной версии сайта, `Vary` не позволит кэшу отдать мобильную версию десктопному браузеру.
*   **Интернационализация**: `Vary: Accept-Language`. Если язык страницы зависит от заголовка `Accept-Language`, `Vary` обеспечивает хранение разных языковых версий в кэше.
*   **CORS**: `Vary: Origin`. При работе с CORS кэшу необходимо различать ответы для разных источников (origins).

# Проблемы и особенности

*   **Раздувание кэша**: Слишком много заголовков в `Vary` (или заголовки с большим количеством уникальных значений, как `User-Agent`) могут привести к резкому снижению эффективности кэширования, так как для каждого уникального набора значений будет создаваться отдельная запись в кэше.
*   **`Vary: *`**: Специальное значение `*` означает, что ответ уникален для каждого запроса (возможно, зависит от не-HTTP факторов, например, IP-адреса). Это фактически запрещает кэширование ответа общими кэшами.
*   **Недооценка важности**: Отсутствие `Vary: Accept-Encoding` при использовании сжатия является распространенной ошибкой, которая может привести к тому, что некоторые пользователи получат "мусор" вместо контента.

[[Caching]]

Заголовок запроса `If-Match` используется для условных запросов. Он прямо противоположен `If-None-Match`. Запрос (например, `PUT` или `PATCH`) будет выполнен сервером только в том случае, если `ETag` ресурса на сервере совпадает с одним из значений, переданных в `If-Match`.

# Описание

Клиент, который собирается обновить или удалить ресурс, может передать `ETag` этого ресурса, который он получил ранее.

`If-Match: "33a64df551425fcc55e4d42a148795d9f25f89d4"`

Если текущий `ETag` на сервере не совпадает с указанным, это означает, что ресурс был изменен кем-то другим с тех пор, как клиент его получил. В этом случае сервер отклонит запрос с кодом `412 Precondition Failed`.

# Юзкейсы

*   **Предотвращение "потерянных обновлений" (Optimistic Locking)**: Это основной юзкейс.
    1.  Клиент A запрашивает ресурс и получает `ETag: "v1"`.
    2.  Клиент B запрашивает тот же ресурс и тоже получает `ETag: "v1"`.
    3.  Клиент B обновляет ресурс. Сервер сохраняет изменения и устанавливает новый `ETag: "v2"`.
    4.  Клиент A пытается обновить ресурс, отправляя свои изменения с заголовком `If-Match: "v1"`.
    5.  Сервер видит, что `ETag` не совпадает (`"v2"` != `"v1"`), и отклоняет запрос клиента A, предотвращая затирание изменений клиента B.
*   **Синхронизация данных**: Гарантирует, что клиент модифицирует именно ту версию ресурса, которую он намеревался изменить.
*   **Частичные запросы**: `If-Match` может использоваться вместе с `Range` для запроса части большого файла, чтобы убедиться, что файл не изменился во время скачивания.

# Проблемы и особенности

*   **`If-Match: *`**: Специальное значение `*` означает, что операция должна быть выполнена только в том случае, если ресурс существует на сервере (независимо от его `ETag`).
*   **Требует поддержки `ETag`**: Для использования `If-Match` сервер должен генерировать и отправлять `ETag` для своих ресурсов.
*   **Сложность для клиента**: Клиентское приложение должно быть спроектировано так, чтобы корректно обрабатывать ответы `412 Precondition Failed`, например, запрашивая свежую версию ресурса и предлагая пользователю разрешить конфликт.

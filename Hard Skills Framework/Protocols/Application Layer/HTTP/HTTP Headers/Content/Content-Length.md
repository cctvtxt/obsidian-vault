[[Content]]

Заголовок `Content-Length` указывает размер тела сообщения в октетах (байтах).

# Описание

Это один из самых простых заголовков. Его значение — целое неотрицательное число.

`Content-Length: 3495`

*   В **запросах** (`POST`, `PUT`) `Content-Length` сообщает серверу, сколько данных ожидать в теле.
*   В **ответах** он сообщает клиенту (браузеру), сколько данных нужно загрузить.

# Юзкейсы

*   **Индикация прогресса загрузки**: Браузеры используют `Content-Length` для отображения прогресс-бара при скачивании файлов. Без него невозможно определить общий размер и показать прогресс.
*   **Проверка целостности**: Получатель может сравнить `Content-Length` с реальным количеством полученных байт, чтобы убедиться, что сообщение не было "обрезано" при передаче.
*   **Управление соединениями**: Веб-серверы и прокси используют `Content-Length` для определения конца одного сообщения и начала следующего в рамках одного keep-alive соединения.

# Проблемы и особенности

*   **Обязателен для некоторых методов**: Для `POST` и `PUT` запросов без `Transfer-Encoding: chunked` заголовок `Content-Length` является обязательным, чтобы сервер знал, где заканчивается тело запроса.
*   **Несовместимость с `Transfer-Encoding: chunked`**: Заголовок `Content-Length` **не должен** использоваться, если применяется `Transfer-Encoding: chunked`. Эти два механизма определения длины тела являются взаимоисключающими. `chunked` имеет приоритет.
*   **Неправильное значение**: Если `Content-Length` больше реального размера тела, получатель будет ждать недостающие данные до тайм-аута. Если меньше — соединение может быть преждевременно закрыто, и данные будут получены не полностью.
*   **Сжатие**: `Content-Length` должен указывать размер тела **после** сжатия (если используется `Content-Encoding`), то есть фактический размер передаваемых данных.

[[Content]]

Заголовок запроса `Accept-Language` сообщает серверу, на каких человеческих языках клиент предпочитает получать ответ. Это еще один механизм согласования контента (*content negotiation*).

# Описание

Значение — это список из одного или нескольких языковых тегов (language tags), определенных в BCP 47. Можно использовать q-факторы для указания предпочтений.

`Accept-Language: ru-RU, ru;q=0.9, en-US;q=0.8, en;q=0.7`

В этом примере клиент говорит: "Больше всего я хочу контент на русском языке (вариант для России). Просто русский тоже отлично (`q=0.9`). Американский английский тоже сойдет (`q=0.8`), и в крайнем случае любой английский (`q=0.7`)".

*   `ru-RU`: Язык `ru` (русский), регион `RU` (Россия).
*   `en-US`: Язык `en` (английский), регион `US` (США).
*   `*`: Может использоваться для обозначения любого языка.

# Юзкейсы

*   **Мультиязычные сайты**: Веб-сервер может анализировать `Accept-Language` и автоматически отдавать пользователю версию сайта на его родном языке, если такая версия существует.
*   **Локализация формата дат и чисел**: На основе языка сервер может форматировать даты, время, числа и валюту в соответствии с локалью пользователя.
*   **Гео-таргетинг**: Хотя для этого лучше подходят другие механизмы (например, IP-адрес), `Accept-Language` может служить дополнительным сигналом о местоположении пользователя.

# Проблемы и особенности

*   **Настройки браузера**: Значение этого заголовка определяется настройками языка в браузере или операционной системе пользователя. Часто пользователи не меняют эти настройки, и они могут не соответствовать их реальным предпочтениям.
*   **Игнорирование сервером**: Как и в случае с `Accept`, многие серверы игнорируют этот заголовок. Большинство сайтов предпочитают давать пользователю явную возможность переключить язык через интерфейс.
*   **`Vary`**: Если сервер использует `Accept-Language` для формирования ответа, он **обязан** включить `Vary: Accept-Language` в ответ, чтобы кэширующие прокси хранили разные языковые версии отдельно.
*   **Ответ `406 Not Acceptable`**: Если сервер не может предоставить контент ни на одном из запрошенных языков, он может вернуть `406 Not Acceptable`.

[[Content]]

Заголовок запроса `Accept-Encoding` сообщает серверу, какие алгоритмы сжатия контента (`Content-Encoding`) клиент понимает и может обработать.

# Описание

Значением является список из одного или нескольких алгоритмов сжатия, разделенных запятыми. Как и в `Accept`, можно использовать q-факторы для указания предпочтений.

`Accept-Encoding: gzip, deflate, br;q=1.0, *;q=0.5`

В этом примере клиент говорит: "Я поддерживаю сжатие `gzip`, `deflate` и `Brotli` (причем `Brotli` предпочтительнее всего). Если ты умеешь какой-то другой алгоритм (`*`), я, возможно, тоже его пойму (`q=0.5`)".

**Основные значения:**

*   `gzip`: Самый распространенный и поддерживаемый алгоритм.
*   `br`: Brotli, более эффективный алгоритм, поддерживаемый современными браузерами.
*   `deflate`: Менее распространенный, чем `gzip`.
*   `identity`: Означает отсутствие сжатия. Это значение всегда считается приемлемым, даже если не указано явно.

# Юзкейсы

*   **Эффективное сжатие**: Это ключевой заголовок для веб-производительности. Браузер отправляет его, чтобы сервер мог сжать ответ и ускорить его доставку. Современные браузеры отправляют `gzip, deflate, br`, позволяя серверу выбрать наилучший из доступных алгоритмов.
*   **Отключение сжатия**: В редких случаях (например, при отладке или работе через криво настроенный прокси) может потребоваться запросить ресурс без сжатия: `Accept-Encoding: identity`.

# Проблемы и особенности

*   **Обязательная поддержка `identity`**: Сервер всегда должен быть готов отдать ресурс без сжатия (`identity`), если клиент не указал ни одного поддерживаемого алгоритма или запросил `identity`.
*   **Неправильная конфигурация сервера**: Если сервер отправляет сжатый контент клиенту, который не указал соответствующий алгоритм в `Accept-Encoding`, клиент не сможет распаковать ответ и отобразит ошибку.
*   **Прокси-серверы**: Промежуточные прокси могут изменять `Accept-Encoding` или кэшировать ответы, сжатые определенным образом. Заголовок `Vary: Accept-Encoding` в ответе сервера важен, чтобы прокси хранил разные сжатые версии для разных клиентов.

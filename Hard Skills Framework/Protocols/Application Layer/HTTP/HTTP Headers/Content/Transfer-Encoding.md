[[Content]]

Заголовок `Transfer-Encoding` указывает, какой тип кодирования был применен для безопасной передачи тела сообщения от отправителя к получателю. Он относится к передаче данных, а не к самому контенту.

# Описание

Основное и практически единственное используемое сегодня значение — `chunked`.

`Transfer-Encoding: chunked`

Это означает, что тело сообщения будет передано в виде последовательности "чанков" (кусков) данных. Каждый чанк состоит из:
1.  Размера чанка в шестнадцатеричном формате, за которым следует `
`.
2.  Данных чанка, за которыми следует `
`.

Передача заканчивается последним чанком нулевой длины (`0

`).

# Юзкейсы

*   **Динамический контент неизвестной длины**: Это основной юзкейс. Когда сервер генерирует ответ на лету (например, результат долгого SQL-запроса) и не знает заранее его итоговый размер, он не может использовать заголовок `Content-Length`. Вместо этого он отправляет `Transfer-Encoding: chunked` и начинает передавать данные по мере их готовности.
*   **Потоковая передача (Streaming)**: `chunked` кодирование идеально подходит для стриминга видео или аудио, когда контент генерируется или передается в реальном времени.

# Проблемы и особенности

*   **Взаимоисключение с `Content-Length`**: Заголовок `Transfer-Encoding` (со значением `chunked`) и `Content-Length` не могут использоваться одновременно в одном сообщении. Если присутствуют оба, `Transfer-Encoding` имеет приоритет.
*   **Только для HTTP/1.1**: `chunked` кодирование было введено в HTTP/1.1. Старые клиенты HTTP/1.0 его не поддерживают.
*   **Устарело в HTTP/2+**: В HTTP/2 и более новых версиях протокола появился свой, более эффективный механизм потоковой передачи данных (фреймы). `Transfer-Encoding: chunked` там больше не используется и запрещен. Заголовок может присутствовать, но должен содержать только `trailers`.
*   **`trailers`**: `chunked` кодирование позволяет передавать заголовки не только в начале, но и в конце сообщения (trailers). Это может быть полезно, если значение какого-то заголовка (например, цифровая подпись контента) можно вычислить только после полной генерации тела. Для этого используется заголовок `Trailer` в начале ответа, перечисляющий имена заголовков, которые придут в конце.

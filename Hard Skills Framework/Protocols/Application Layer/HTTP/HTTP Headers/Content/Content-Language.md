[[Content]]

Заголовок ответа `Content-Language` описывает человеческий язык (или языки), для которого предназначен контент. Он **не** указывает язык самого контента, а скорее его целевую аудиторию.

# Описание

Значением является один или несколько языковых тегов.

`Content-Language: de-DE`
`Content-Language: en-US, ru-RU`

Этот заголовок позволяет пользователю понять, для какой языковой аудитории предназначен документ. Например, учебник по английскому языку, предназначенный для русскоязычной аудитории, может быть помечен как `Content-Language: ru`, хотя сам текст будет на английском.

# Юзкейсы

*   **Индексация поисковыми системами**: Помогает поисковым системам правильно классифицировать контент и показывать его релевантной языковой аудитории.
*   **Выбор правильного перевода**: Если ресурс доступен на нескольких языках, этот заголовок может помочь браузеру или приложению предложить пользователю правильную версию.

# Проблемы и особенности

*   **Путаница с `lang`**: Не следует путать `Content-Language` с атрибутом `lang` в HTML (`<html lang="en">`). `lang` описывает язык конкретного элемента, тогда как `Content-Language` описывает язык аудитории для всего документа. Они могут не совпадать.
*   **Не указывает язык контента**: Распространенное заблуждение — считать, что этот заголовок описывает язык текста. Он описывает аудиторию. Хотя в большинстве случаев они совпадают, это не является правилом.
*   **`Accept-Language`**: `Content-Language` не связан напрямую с заголовком запроса `Accept-Language`. Сервер может вернуть контент с `Content-Language: en-US` даже если клиент запрашивал `Accept-Language: fr`.
*   **Несколько языков**: Если контент предназначен для нескольких аудиторий, можно перечислить несколько языков через запятую.

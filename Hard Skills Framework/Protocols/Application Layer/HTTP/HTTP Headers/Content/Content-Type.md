[[Content]]

Заголовок ответа (и запроса) `Content-Type` указывает на медиатип (MIME-тип) ресурса в теле сообщения. Он сообщает получателю, как интерпретировать данные.

# Описание

`Content-Type` является одним из самых фундаментальных заголовков в HTTP. Его значение состоит из типа, подтипа и опциональных параметров (например, кодировки).

`Content-Type: type/subtype; charset=utf-8`

**Примеры:**

*   `Content-Type: text/html; charset=utf-8` — HTML-документ в кодировке UTF-8.
*   `Content-Type: application/json` — Данные в формате JSON.
*   `Content-Type: image/jpeg` — Изображение в формате JPEG.
*   `Content-Type: multipart/form-data; boundary=something` — Данные формы, состоящие из нескольких частей (например, при загрузке файлов).

# Юзкейсы

*   **Отображение в браузере**: Браузер использует `Content-Type` ответа, чтобы понять, как отобразить контент: как HTML-страницу, как картинку, или предложить скачать файл (`application/octet-stream`).
*   **Обработка данных API**: В REST API `Content-Type: application/json` в запросе сообщает серверу, что в теле находится JSON. Сервер, в свою очередь, отвечает с таким же `Content-Type`, чтобы клиент знал, как парсить ответ.
*   **Загрузка файлов**: При отправке файлов через форму используется `multipart/form-data`, который позволяет передавать бинарные данные.
*   **Безопасность**: Установка `Content-Type` вместе с заголовком `X-Content-Type-Options: nosniff` предотвращает MIME-сниффинг атаки, при которых браузер пытается "угадать" тип контента, что может привести к уязвимостям.

# Проблемы и особенности

*   **Отсутствие заголовка**: Если `Content-Type` не указан, получатель может попытаться угадать его (MIME sniffing), что небезопасно. По умолчанию многие системы считают данные `application/octet-stream` (бинарный поток).
*   **Неправильный `Content-Type`**: Отправка данных с неверным MIME-типом может привести к тому, что браузер или клиентское приложение не сможет их корректно обработать. Например, отправка JSON с типом `text/plain`.
*   **`charset`**: Явное указание кодировки (`charset=utf-8`) является хорошей практикой, особенно для текстовых форматов, чтобы избежать проблем с отображением символов.

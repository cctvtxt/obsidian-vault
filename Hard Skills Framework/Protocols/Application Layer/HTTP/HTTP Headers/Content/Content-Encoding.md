[[Content]]

Заголовок `Content-Encoding` указывает, какие кодировки (обычно алгоритмы сжатия) были применены к телу сообщения. Получатель должен применить обратные преобразования в указанном порядке, чтобы получить исходные данные.

# Описание

Значением является одно или несколько имен алгоритмов, разделенных запятыми.

`Content-Encoding: gzip`
`Content-Encoding: br`
`Content-Encoding: gzip, deflate`

**Основные значения:**

1.  `gzip`: Сжатие с использованием Lempel-Ziv coding (LZ77) и 32-битного CRC. Самый распространенный и поддерживаемый метод.
2.  `compress`: Алгоритм Lempel-Ziv-Welch (LZW). Устарел и практически не используется.
3.  `deflate`: Сжатие с использованием zlib-структуры и deflate-алгоритма. Менее популярен, чем `gzip`.
4.  `br`: Brotli — современный алгоритм сжатия, обеспечивающий лучшую степень сжатия, чем `gzip`, особенно для текста. Широко поддерживается современными браузерами.
5.  `identity`: Указывает на отсутствие сжатия. Используется по умолчанию, если заголовок отсутствует.

# Юзкейсы

*   **Ускорение загрузки**: Сжатие HTML, CSS, JS и JSON ответов значительно уменьшает их размер, что сокращает время загрузки, особенно на медленных соединениях. Это стандартная практика для веб-производительности.
*   **Экономия трафика**: Уменьшение размера передаваемых данных экономит трафик как для пользователя, так и для сервера.

# Проблемы и особенности

*   **Согласование с клиентом**: Сервер не должен применять сжатие, если клиент его не поддерживает. Для этого используется заголовок запроса `Accept-Encoding`, в котором клиент перечисляет поддерживаемые им алгоритмы.
*   **`Content-Length`**: Заголовок `Content-Length` должен указывать размер тела **после** применения `Content-Encoding`.
*   **Нагрузка на CPU**: Сжатие на лету потребляет ресурсы процессора. Для статических файлов их обычно сжимают заранее и хранят сжатые копии. Brotli требует больше ресурсов для сжатия, чем `gzip`, но обеспечивает лучшую компрессию.
*   **Не для всего**: Сжимать уже сжатые форматы (JPEG, PNG, MP4, PDF) неэффективно и является пустой тратой ресурсов CPU. Сжатие применяется в основном к текстовым данным.

[[HTTP Headers]]

**Пользовательские (кастомные) заголовки**, часто с префиксом `X-`, используются для передачи специфичной для приложения информации, которая не покрывается стандартными заголовками. Они позволяют разработчикам расширять протокол HTTP для своих нужд, например, для отладки, передачи уникальных идентификаторов или дополнительной метаинформации. Хотя стандарт `RFC 6648` рекомендует избегать префикса `X-` в пользу создания новых стандартных заголовков, он все еще широко распространен.

1. `X-Request-ID` — уникальный идентификатор для отслеживания запроса.
2. `X-Forwarded-For` — IP-адрес клиента, прошедшего через прокси.
3. `X-API-Key` — ключ для доступа к API.
4. `X-CSRF-Token` — токен для защиты от CSRF-атак.
5. `X-Frame-Options` — защита от кликджекинга.
6. `X-Content-Type-Options` — предотвращает MIME-сниффинг.
7. `X-XSS-Protection` — включает фильтры XSS в браузерах.

[[HTTP Headers]]

Заголовки **маршрутизации** используются для управления тем, как запрос проходит через сеть, особенно через прокси-серверы и балансировщики нагрузки. Они предоставляют информацию об исходном клиенте и о пути, который прошел запрос.

1. `Host` — целевой хост виртуального сервера.
2. `Forwarded` — стандартный способ передать данные о прокси-цепочке.
3. `X-Forwarded-For` — исходный IP клиента через прокси.
4. `X-Forwarded-Proto` — исходный протокол, например `http` или `https`.
5. `X-Forwarded-Host` — исходный host до прокси.
6. `X-Real-IP` — реальный IP клиента, переданный прокси.
7. `X-Client-IP` — альтернативный заголовок с IP клиента.
8. `Via` — показывает промежуточные прокси.
9. `Max-Forwards` — ограничивает число прокси-переходов.
10. `Proxy-Connection` — нестандартный заголовок, иногда используется прокси.

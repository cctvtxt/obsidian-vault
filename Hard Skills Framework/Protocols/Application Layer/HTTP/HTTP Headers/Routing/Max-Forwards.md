[[Hard Skills Framework/Protocols/Application Layer/HTTP/HTTP Headers/Routing]]

Заголовок запроса `Max-Forwards` используется с методами `TRACE` и `OPTIONS` для ограничения количества прокси-серверов или шлюзов, через которые может пройти запрос.

# Описание

Значением является целое число, которое указывает, сколько еще "прыжков" (hops) может сделать запрос.

`Max-Forwards: 10`

Каждый прокси-сервер, получив запрос с этим заголовком, должен уменьшить его значение на единицу перед тем, как отправить запрос дальше.

Если прокси получает запрос с `Max-Forwards: 0`, он не должен пересылать его дальше, а обязан ответить на него самостоятельно.

# Юзкейсы

*   **Отладка цепочки прокси**: Основное предназначение `Max-Forwards` — это отладка. С помощью метода `TRACE` клиент может увидеть, как его запрос изменяется на пути к конечному серверу.
    *   Отправляя `TRACE` с `Max-Forwards: 0`, клиент получит ответ от первого же прокси на пути.
    *   С `Max-Forwards: 1` — от второго, и так далее.
    Это позволяет пошагово исследовать всю цепочку прокси.
*   **Диагностика с `OPTIONS`**: `OPTIONS` запрашивает информацию о возможностях следующего узла в цепочке. Используя `Max-Forwards`, можно запросить информацию о возможностях конкретного прокси, а не конечного сервера.

# Проблемы и особенности

*   **Ограниченное применение**: Этот заголовок используется только с `TRACE` и `OPTIONS` и не влияет на другие методы, такие как `GET` или `POST`.
*   **`TRACE` отключен**: Метод `TRACE` часто отключают на серверах по соображениям безопасности (может использоваться для атак типа Cross-Site Tracing), что снижает практическую пользу `Max-Forwards`.
*   **Не является механизмом TTL**: `Max-Forwards` не следует путать с TTL (Time To Live) в IP-пакетах. Он не предотвращает бесконечные циклы для всех типов запросов, а служит лишь для диагностики с `TRACE`/`OPTIONS`. Для обнаружения циклов используется заголовок `Via`.

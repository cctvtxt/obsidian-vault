[[Hard Skills Framework/Protocols/Application Layer/HTTP/HTTP Headers/Routing]]

Заголовок `Via` добавляется прокси-серверами (как прямыми, так и обратными) и шлюзами для указания того, что они участвовали в обработке запроса. Он используется для отслеживания пути сообщения и предотвращения зацикливания запросов.

# Описание

Каждый прокси на пути запроса добавляет себя в заголовок `Via`. Заголовок может содержать несколько записей, разделенных запятыми.

`Via: <HTTP-version> <proxy-name-or-identifier> (<comment>)`

Пример:
`Via: 1.1 vegur, 1.1 varnish`
`Via: HTTP/1.1 GWA, 1.0 fred`

*   **В запросе**: Показывает путь от клиента к серверу.
*   **В ответе**: Показывает обратный путь от сервера к клиенту.

# Юзкейсы

*   **Обнаружение зацикливания запросов**: Если прокси-сервер получает запрос, в котором он уже присутствует в заголовке `Via`, он понимает, что запрос зациклился, и должен прервать его (например, с ошибкой `508 Loop Detected`).
*   **Отладка маршрутизации**: `Via` позволяет администраторам и разработчикам видеть, через какие промежуточные узлы проходит трафик.
*   **Информация о протоколах**: Заголовок указывает версию протокола, которую использовал каждый прокси. Это может быть полезно для диагностики проблем совместимости.

# Проблемы и особенности

*   **Раскрытие информации**: `Via` может раскрывать информацию о внутренней архитектуре сети (например, имена и версии прокси-серверов), что может быть нежелательно с точки зрения безопасности. Некоторые прокси могут быть настроены так, чтобы изменять или удалять этот заголовок.
*   **Идентификатор прокси**: Спецификация не требует, чтобы идентификатор прокси был реальным хостнеймом. Это может быть псевдоним.
*   **Отличие от `Forwarded`**: `Via` сообщает о самих прокси, в то время как `Forwarded` (и `X-Forwarded-For`) сообщает об исходном клиенте. Они служат разным, хотя и связанным, целям.

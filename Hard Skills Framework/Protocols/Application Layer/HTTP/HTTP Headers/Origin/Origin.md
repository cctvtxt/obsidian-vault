[[Hard Skills Framework/Protocols/Application Layer/HTTP/HTTP Headers/Origin]]

Заголовок запроса `Origin` указывает источник (домен, схему и порт), с которого исходит кросс-доменный запрос. Он является ключевым элементом в механизме безопасности CORS.

# Описание

Когда JavaScript-код со страницы `https://example.com` делает запрос (например, `fetch`) к другому домену `https://api.service.com`, браузер автоматически добавляет к этому запросу заголовок:

`Origin: https://example.com`

Этот заголовок информирует сервер `api.service.com` о том, откуда пришел запрос. На основе этого значения сервер решает, можно ли доверять этому источнику и разрешать ли ему доступ к ресурсу.

Значение `Origin` может быть `null` для запросов, источник которых невозможно определить (например, из локального `file://` URI).

# Юзкейсы

*   **Основа CORS**: `Origin` является отправной точкой для всего механизма CORS. Сервер смотрит на этот заголовок и на его основе формирует ответные CORS-заголовки (например, `Access-Control-Allow-Origin`), разрешая или отклоняя запрос.
*   **Безопасность**: Позволяет серверу реализовать политику "белого списка" доменов, которым разрешено обращаться к его API из браузера. Все запросы с `Origin`, не входящих в этот список, будут заблокированы браузером.

# Проблемы и особенности

*   **Добавляется браузером**: Этот заголовок добавляется и контролируется браузером. Его нельзя изменить из JavaScript. Это гарантирует, что сервер получает достоверную (с точки зрения браузера) информацию об источнике запроса.
*   **Не во всех запросах**: `Origin` добавляется только к кросс-доменным запросам и к same-origin запросам с методами `POST`, `PUT`, `DELETE`. Обычные `GET` запросы в пределах одного домена могут не содержать `Origin`.
*   **`Origin: null`**: Запрос с `Origin: null` представляет особый случай. Сервер должен решить, разрешать ли доступ анонимным источникам. Часто такие запросы блокируются.
*   **Не только для `fetch`/`XHR`**: Заголовок `Origin` также отправляется при кросс-доменных запросах на шрифты, изображения в `<canvas>` и другие ресурсы, доступ к которым ограничен политикой безопасности.

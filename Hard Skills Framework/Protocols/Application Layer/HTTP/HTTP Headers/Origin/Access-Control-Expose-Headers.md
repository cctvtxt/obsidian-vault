`Access-Control-Expose-Headers` — это заголовок ответа, который указывает, какие из заголовков ответа могут быть доступны клиентскому JavaScript-коду.

# Описание

По соображениям безопасности, браузер по умолчанию предоставляет JavaScript-коду доступ только к "простым" заголовкам ответа:
*   `Cache-Control`
*   `Content-Language`
*   `Content-Length`
*   `Content-Type`
*   `Expires`
*   `Last-Modified`
*   `Pragma`

Если сервер хочет сделать доступными другие заголовки (например, кастомные, как `X-My-Custom-Header`, или стандартные, как `ETag`), он должен перечислить их в `Access-Control-Expose-Headers`.

`Access-Control-Expose-Headers: Content-Disposition, ETag, X-Pagination-Count`

# Юзкейсы

*   **Получение метаданных пагинации**: API в ответ на запрос списка элементов может возвращать общее количество элементов в кастомном заголовке (например, `X-Total-Count`). Чтобы клиент мог прочитать этот заголовок, сервер должен его "раскрыть".
*   **Чтение `ETag`**: Для реализации логики оптимистичных блокировок (`If-Match`) клиенту нужно сначала прочитать `ETag` из ответа на `GET`-запрос.
*   **Чтение имени файла**: При скачивании файлов сервер часто отправляет имя файла в заголовке `Content-Disposition`. Чтобы JavaScript мог его прочитать и использовать (например, для сохранения файла с правильным именем), этот заголовок должен быть раскрыт.

# Проблемы и особенности

*   **Безопасность**: Раскрывайте только те заголовки, которые действительно нужны клиентскому коду. Раскрытие лишних заголовков может привести к утечке чувствительной информации.
*   **`*` (wildcard)**: Использование `*` не стандартизировано и не рекомендуется. Необходимо явно перечислять все заголовки.
*   **Не нужен для "простых" заголовков**: Перечислять в этом заголовке "простые" заголовки (вроде `Content-Type`) не нужно, они доступны по умолчанию.

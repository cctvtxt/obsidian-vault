`Access-Control-Request-Method` — это заголовок запроса, который браузер использует в preflight-запросе (`OPTIONS`), чтобы сообщить серверу, какой HTTP-метод будет использован в фактическом, последующем запросе.

# Описание

Когда JavaScript-код пытается выполнить кросс-доменный запрос с использованием "непростого" метода (например, `PUT` или `DELETE`), браузер перехватывает этот вызов. Вместо того чтобы сразу отправлять `PUT` запрос, он сначала отправляет `OPTIONS` запрос к тому же URL и включает в него заголовок `Access-Control-Request-Method`.

`Access-Control-Request-Method: PUT`

Этот заголовок, по сути, является вопросом к серверу: "Собираюсь отправить `PUT` запрос к этому ресурсу. Это разрешено?".

Сервер должен проанализировать этот заголовок и ответить, разрешен ли данный метод, используя заголовок ответа `Access-Control-Allow-Methods`.

# Юзкейсы

*   **Часть механизма Preflight**: Этот заголовок является неотъемлемой частью preflight-проверки в CORS. Он позволяет серверу заранее узнать о намерениях клиента и принять решение о разрешении или запрете действия до того, как основной, потенциально "опасный" запрос будет отправлен.

# Проблемы и особенности

*   **Добавляется браузером**: Этот заголовок полностью контролируется браузером. Его нельзя установить или изменить из JavaScript.
*   **Только в `OPTIONS`**: `Access-Control-Request-Method` отправляется **только** в preflight-запросах, которые используют метод `OPTIONS`.
*   **Один метод**: В отличие от ответного заголовка `Access-Control-Allow-Methods`, который может содержать список методов, этот заголовок всегда содержит только один метод — тот, который будет использован в следующем запросе.
*   **"Непростые" методы**: Preflight-проверка с этим заголовком инициируется для всех методов, кроме `GET`, `HEAD` и `POST` (при условии, что `Content-Type` у `POST` тоже "простой").

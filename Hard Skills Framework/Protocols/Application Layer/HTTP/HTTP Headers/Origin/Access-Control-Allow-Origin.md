[[Hard Skills Framework/Protocols/Application Layer/HTTP/HTTP Headers/Origin]]

`Access-Control-Allow-Origin` — это заголовок ответа, который сообщает браузеру, разрешено ли коду с указанного источника (`Origin`) получать доступ к ответу. Это самый главный заголовок в механизме CORS.

# Описание

Когда сервер получает кросс-доменный запрос с заголовком `Origin`, он должен включить в ответ заголовок `Access-Control-Allow-Origin`, чтобы разрешить доступ.

**Возможные значения:**

1.  **Конкретный источник**: `Access-Control-Allow-Origin: https://example.com`
    Это самое безопасное значение. Оно говорит браузеру: "Коду с `https://example.com` можно читать этот ответ". Если `Origin` запроса был другим, браузер заблокирует ответ.

2.  **Любой источник**: `Access-Control-Allow-Origin: *`
    Это разрешает доступ к ответу коду с абсолютно любого домена. Используется для публичных API, которые не содержат чувствительных данных и могут быть вызваны с любого сайта.

3.  **`null`**: `Access-Control-Allow-Origin: null`
    Разрешает доступ для запросов с `Origin: null`.

# Юзкейсы

*   **Разрешение доступа к API**: Основная задача — разрешить фронтенд-приложению, размещенному на одном домене, обращаться к бэкенду на другом домене.
*   **Публичные ресурсы**: Разрешить встраивание и использование публичных ресурсов (шрифтов, данных) на любых сайтах с помощью `*`.

# Проблемы и особенности

*   **Динамическое значение**: Сервер не может указать список доменов в одном заголовке. Если нужно поддерживать несколько источников, серверная логика должна проверять `Origin` запроса и, если он есть в "белом списке", подставлять его в `Access-Control-Allow-Origin`.
*   **`*` и `credentials`**: Использование `Access-Control-Allow-Origin: *` **несовместимо** с передачей учетных данных (cookie, HTTP-аутентификация). Если в запросе был `withCredentials: true`, то `AC-Allow-Origin` должен быть указан явно (например, `https://example.com`), а не `*`.
*   **Блокировка браузером**: Если сервер не возвращает этот заголовок, или его значение не совпадает с `Origin` запроса (и не равно `*`), браузер заблокирует доступ к ответу, и в JavaScript будет выброшена ошибка, хотя сам сетевой запрос на сервер прошел успешно.

#Project

**.eslintrc**: Конфигурационный файл для ESLint, определяющий правила linting (проверки стиля, ошибок и лучших практик в JS/TS коде)

```json
{
  "root": true,
  // Указывает, что это корневой файл конфигурации ESLint, ESLint прекращает поиск файлов .eslintrc в родительских директориях
  "parser": "@typescript-eslint/parser",
  // используется `@typescript-eslint/parser`, который позволяет ESLint корректно разбирать код на TypeScript.
  "env": { "node": true },
  // Задаёт среду выполнения кода. Установка `"node": true` включает глобальные переменные, специфичные для Node.js (например, `process`, `module`), и предполагает, что код работает в среде Node.js
  "plugins": ["@typescript-eslint"],
  // Плагин `@typescript-eslint` предоставляет правила линтинга, специфичные для TypeScript, поддержку  TypeScript-кода, дополняя парсер
  "extends": [
    "eslint:recommended",
    // Наследует набор рекомендованных правил из ядра ESLint. Это разумные настройки по умолчанию для обнаружения распространённых ошибок в JavaScript (например, неопределённые переменные, пропущенные точки с запятой).
    "plugin:@typescript-eslint/eslint-recommended",
    // Наследует конфигурацию от плагина `@typescript-eslint`, которая отключает или изменяет правила ESLint, конфликтующие с TypeScript, чтобы избежать ненужных ошибок.
    "plugin:@typescript-eslint/recommended"
    // Включает рекомендованные правила линтинга от плагина `@typescript-eslint`, которые обеспечивают проверку TypeScript-кода на распространённые ошибки и лучшие практики.
  ],
  "parserOptions": {
    "sourceType": "module"
    // Указывает, что код использует синтаксис ES-модулей (import/export). Это важно для правильной обработки модульного кода в TypeScript и JavaScript.
  },
  "rules": {
    "no-unused-vars": "off",
    // Отключает стандартное правило ESLint `no-unused-vars`, которое проверяет наличие неиспользуемых переменных, чтобы вместо него использовать TypeScript-специфичную версию правила
    "@typescript-eslint/no-unused-vars": ["error", { "args": "none" }],
    // Включает правило `@typescript-eslint/no-unused-vars` с уровнем строгости "error". Опция `"args": "none"` означает, что неиспользуемые аргументы функций не будут считаться ошибками, но переменные будут
    "@typescript-eslint/ban-ts-comment": "off",
    // Отключает правило, которое запрещает использование комментариев `@ts-<directive>` (например, `@ts-ignore`)
    "no-prototype-builtins": "off",
    // Отключает правило ESLint, которое запрещает прямое использование встроенных методов прототипов (например, `hasOwnProperty`)
    "@typescript-eslint/no-empty-function": "off"
    // Отключает правило, которое запрещает пустые функции в TypeScript
  }
}
```

**esbuild.config.mjs**: Скрипт конфигурации для esbuild. Ключевой для сборки плагина Obsidian: преобразует main.ts в main.js

```json
    "scripts": {
        "dev": "node esbuild.config.mjs",
        "build": "tsc -noEmit -skipLibCheck && node esbuild.config.mjs production",
        "version": "node version-bump.mjs && git add manifest.json versions.json"
    },
```

```json
import esbuild from "esbuild";
// esbuild — быстрый бандлер и компилятор для JavaScript/TypeScript. Это позволяет использовать API esbuild для сборки кода.
import process from "process";
// "process", который предоставляет информацию о текущем процессе (например, аргументы командной строки через process.argv).
import builtins from "builtin-modules";
// "builtin-modules", которая возвращает массив встроенных модулей Node.js ('fs', 'path' b nl). Это используется для исключения их из бандла

const banner =
`/*
THIS IS A GENERATED/BUNDLED FILE BY ESBUILD
if you want to view the source, please visit the github repository of this plugin
*/
`;

// Определяет константу banner как многострочную строку (шаблонный литерал). Это текст, который будет добавлен в начало сгенерированного JS-файла для информационных целей.

const prod = (process.argv[2] === "production");
// Определяет prod
const context = await esbuild.context({
//  Контекст позволяет повторно использовать настройки для нескольких сборок или наблюдения.

banner: {
js: banner,
},

// Устанавливает баннер для JS-файла: добавляет текст из переменной banner в начало выходного файла.
entryPoints: ["main.ts"],

// Указывает точки входа: файл "main.ts" — это основной файл, с которого начинается сборка (вероятно, входной TypeScript-файл плагина).

bundle: true,

// Включает режим бандлинга: esbuild объединит все зависимости в один файл, кроме указанных как external.

external: [

"obsidian",

"electron",

"@codemirror/autocomplete",

"@codemirror/collab",

"@codemirror/commands",

"@codemirror/language",

"@codemirror/lint",

"@codemirror/search",

"@codemirror/state",

"@codemirror/view",

"@lezer/common",

"@lezer/highlight",

"@lezer/lr",

...builtins],

// Список внешних модулей, которые не включаются в бандл. Они предполагаются доступными в среде выполнения (например, Obsidian предоставляет "obsidian" и Codemirror). ...builtins добавляет все встроенные Node.js модули.

format: "cjs",

// Устанавливает формат вывода как CommonJS (module.exports и require), подходящий для Node.js или Obsidian.

target: "es2018",

// Целевая версия JavaScript: код компилируется до ES2018 (async/await, spread и т.д.), обеспечивая совместимость с браузерами/средами, поддерживающими эту версию.

logLevel: "info",

// Уровень логирования esbuild: "info" — выводит информационные сообщения о процессе сборки (например, время, ошибки).

sourcemap: prod ? false : "inline",

// Устанавливает sourcemap: в продакшене (prod=true) — отключено; иначе — inline (карта источников встраивается в JS для отладки).

treeShaking: true,

// Включает tree shaking: удаляет неиспользуемый код из бандла для оптимизации размера.

outfile: "main.js",

// Имя выходного файла: "main.js" — туда запишется собранный код.

minify: prod,

// Включает минификацию (сжатие кода: удаление пробелов, сокращение имён) только в продакшене (prod=true).

});

if (prod) {

await context.rebuild();

// Если prod=true, выполняет однократную пересборку на основе контекста. Это генерирует main.js с продакшен-настройками.

process.exit(0);

// Завершает процесс Node.js с кодом выхода 0 (успех), чтобы скрипт не висел.

} else {

await context.watch();
// Если prod=false, запускает режим наблюдения: esbuild мониторит изменения в файлах и автоматически пересобирает main.js при обновлениях.
}
```
